## 概述

设计一个游戏内道具商城系统，允许玩家使用游戏货币或真实货币购买虚拟道具。

---

## 用户角色

| 角色 | 权限范围 |
|------|----------|
| **普通玩家** | 浏览商城、购买道具、查看订单 |
| **VIP 玩家** | 享有专属折扣和限时道具（在专属商城中购买 VIP 专属道具） |
| **管理员** | 上下架道具、配置价格、管理促销 |
| **策划** | 配置 VIP 等级规则、促销规则、热门标签（通过配置表/配置后台） |

> **职责边界说明**：策划负责定义业务规则（配置内容），程序负责实现逻辑，测试负责验证程序对配置的消费是否正确。策划不涉及技术实现细节。

## 功能需求

---

### F1: 商城首页

| 项目 | 内容 |
|------|------|
| 热门道具推荐 | 前 10 个，由策划手动标记 `hot` 标签，按策划配置优先级+自然序展示 |
| 分类导航栏 | 武器、时装、坐骑、消耗品、礼包 |
| 搜索功能 | 支持道具名称模糊搜索 |
| 道具列表 | 每页展示 20 个，显示：名称、图标、价格（渠道换算后）、折扣标签 |
| 热门标签管理 | 策划在配置后台为道具打 `hot` 标签，程序读取标签渲染推荐区；hot 标签变更后前端按配置刷新时机展示 |

---

### F2: 道具详情页

| 项目 | 内容 |
|------|------|
| 展示信息 | 道具名称、描述、属性加成、价格（已换算）、折扣标签 |
| 预览功能 | 支持预览（角色穿戴效果） |
| 数量选择 | 购买数量 1-99 |
| 购买按钮 | 余额不足时禁用并显示提示 |

---

### F3: 购买流程

| 项目 | 内容 |
|------|------|
| 确认步骤 | 购买前需确认道具名称、数量、总价 |
| 支付方式 | 游戏币、人民币（道具原价以人民币定义，各渠道程序按汇率表自动换算） |
| 到账时间 | 支付成功后道具立即到账（无延迟） |
| 汇率机制 | 道具价格基准为人民币；渠道（Android/iOS/Web）有各自的汇率配置，结算时程序自动换算；汇率下单时锁定（以订单创建时的汇率为准） |
| 邮件通知 | 购买成功后发送系统邮件，标题和内容由策划定义，走通用邮件配置；发送时机为购买流程完成后；异步发送机制由程序自决策（不限定技术实现） |

**邮件通知内容字段**（策划定义）：
- 玩家名称
- 道具名称
- 购买数量
- 金额（含币种）
- 购买时间
- 订单号

---

### F4: 订单管理

| 项目 | 内容 |
|------|------|
| 记录范围 | 最近 30 天内的购买记录 |
| 记录字段 | 订单号、道具名称、数量、金额（含币种）、支付方式、购买时间 |
| 筛选功能 | 支持按时间范围和道具名称筛选 |

---

### F5: VIP 专属商城

| 项目 | 内容 |
|------|------|
| VIP 等级 | VIP1（95折）、VIP2（9折）、VIP3（85折） |
| 等级来源 | VIP 等级由配置表管理，策划编辑配置表设定各等级解锁条件和权限 |
| 解锁条件 | 各等级有独立解锁条件（配置项，程序预设条件类型：累计消费额/在线时长/持有道具等） |
| 等级时效 | 策划在配置表中设置等级是否有时效（永久 or 定时考核）；当前默认按「配置即有效」处理，不自动续期 |
| 专属道具 | 仅对 VIP 用户可见 |
| 折扣规则 | 在结算时自动计算；**不可叠加**（限时折扣与 VIP 折扣独立存在，取最大值，详见 F6 促销系统） |

---

### F6: 促销系统

| 项目 | 内容 |
|------|------|
| 配置角色 | 策划负责配置，测试负责验收 |
| 配置入口 | 策划工具（配置后台或配置文件） |
| 配置项 | 促销名称、活动时间段、折扣比例/满减规则、适用道具/分类、标签字段 |
| 限时打折 | 通过配置中的折扣标签字段控制（含生效时间段），程序定时检查或事件触发；展示样式由前端根据标签内容渲染（折扣角标、划线原价、倒计时） |
| 满减活动 | 单笔订单满 100 元减 10 元，满 200 元减 25 元 |
| 新手礼包 | 新注册玩家 7 天内可购买一次 |
| 折扣优先级 | 限时折扣标签与 VIP 折扣标签**独立叠加**计算（限时折扣先于 VIP 折扣执行，最终价格为 min(限时折后价, VIP折后价)），或按配置表中的优先级字段决定；待策划确认后以配置为准 |
| 折扣叠加规则（待确认） | 限时折扣与 VIP 折扣叠加逻辑：取最优（最低价）/叠加计算/不可叠加——以策划最终配置为准 |

## 非功能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 2 秒 |
| 购买接口响应时间 | < 500ms |
| 并发支持 | 10000 并发用户 |
| 数据缓存 | 道具数据缓存 5 分钟 |
| 日志 | 所有支付操作需记录日志 |
| 汇率换算精度 | 角分处理遵循四舍五入原则，以分为最小单位 |
| 邮件发送 | 异步队列机制（程序自决策），支持重试 |

## 验收标准

| # | 标准描述 |
|----|----------|
| 1 | 玩家可在商城首页浏览所有分类道具 |
| 2 | 玩家可成功使用游戏币购买道具，到账时间 < 3 秒 |
| 3 | VIP 玩家购买时自动享受对应折扣 |
| 4 | 限时促销期间道具显示折扣标签，价格正确 |
| 5 | 玩家可查看完整购买订单记录 |
| 6 | 余额不足时购买按钮置灰且显示提示 |
| 7 | 热门道具由策划配置 hot 标签后，首页推荐区正确展示（配置变更生效后） |
| 8 | VIP 等级变更后，折扣判定实时生效（以配置表最新数据为准） |
| 9 | 各渠道（Android/iOS/Web）价格换算正确，与人民币基准价一致 |
| 10 | 邮件通知在购买成功后正确发送，内容字段完整 |
