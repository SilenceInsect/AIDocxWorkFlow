# HINT 模块测试点模板（概览）

> **模块代码**：`HINT`
> **来源**：`.cursor/MODULES.md` §4.11
> **作用**：S5 生成 HINT 模块测试点时，按 story 实际涉及子类**按需加载**对应子模板。
> **v1.7 落地**：从 v1.6.1 暂存版 6 枚举（`RED_DOT` / `ITEM_FLOAT` / `CURRENCY_FLOAT` / `MODAL_DIALOG` / `TOAST` / `FLOAT_NOTIFY`）扩展为 13 大类（保留 6 个 + 新增 7 个），覆盖原 v1.6.1 暂存版"原定义完全缺失"的 6 大类场景。
>
> **完整覆盖范围**（一句话）：
> 面向玩家的全局临时可视化反馈：功能入口红点、未读/剩余次数数字角标；道具货币/战斗伤害 buff 动态飘字；操作成功/失败短时 Toast 轻提示；阻断式模态弹窗（错误警告、二次确认、公告、批量奖励汇总）；侧边活动浮动通知、限时倒计时悬浮浮窗；统一规范错误提示文案、活动到期/资源过期提醒；新手引导高亮气泡指引；社交消息、组队/好友申请弹窗；等级晋升/段位结算状态弹窗；防沉迷、付费限额合规预警弹窗；离线补偿、维护补发奖励弹窗；多提示队列管理、层级遮挡校验、查看领取后自动清除提示状态。
> **（页面常驻静态控件、页面内置布局元素归 UI 模块，本模块仅覆盖临时弹出、一次性反馈类提示）**

---

## 1. 核心一句话（项目级 SSoT）

**HINT 聚焦玩家可见的全局临时反馈类提示组件（红点/飘字/Toast/弹窗/浮窗/引导气泡/合规预警/离线补偿弹窗）；页面常驻控件、固定布局、静态页面元素统归 UI 模块——两者无重叠范围。**

---

## 2. HINT vs UI 关键边界隔离规则（S5 误标高发区）

> **核心判断原则**：**临时弹出 = HINT；常驻 = UI**。

| 归 HINT（不归 UI）| 归 UI（不归 HINT）|
| ------------------ | ----------------- |
| 临时弹出、一次性反馈、全局浮动、操作后自动消失的提示组件 | 页面常驻控件、固定布局、页面内置按钮/输入框/分页 |
| 红点、飘字、Toast、弹窗、浮窗、引导气泡 | 静态展示、页面内置文字标签、页面内常驻数值显示 |
| **举例**：背包页面里**固定显示的金币数字** → UI；使用道具弹出**金币+100 飘字** → HINT | **举例**：活动页面**常驻活动标题** → UI；上线自动弹出**活动奖励弹窗** → HINT |
| 邮件未读数角标（**触发弹性的**）| 邮件页面固定列表/分页器/搜索框 |
| 战斗中暴击伤害飘字（**一次性浮现后消失**）| 战斗血条/角色模型/技能图标（**常驻渲染**）|
| 限时活动倒计时浮窗（**事件触发弹出**）| 活动页面常驻倒计时数字（**页面常驻**）|
| 升级弹窗、突破升星弹窗（**事件触发**）| 升级页面常驻控件（**页面常驻**）|
| 新手引导高亮遮罩（**事件触发、点击后消失**）| 引导底层框架（步骤管理器）→ AUX |
| 错误提示文案分级样式（**红黄绿配色**）| 错误码体系、错误处理业务逻辑 → BIZ |

**判定三问**（S5 prompt 强制回答）：
1. **该元素是事件触发弹出，还是页面常驻可见？** 事件触发 → HINT；常驻 → UI
2. **该元素是操作后自动消失，还是玩家手动关闭/长期保留？** 自动消失 → HINT；长期保留 → UI
3. **该测试点是测"显示什么内容/数字/文案"（HINT），还是测"UI 容器的样式/位置/动画"（UI F.GUIDE_HINT）？** 内容 → HINT；样式 → UI

## 4. 加载规则（S5 prompt 使用方式）

1. **检测** `module` 字段 == "HINT"（通常以 `BIZ/UI/CONFIG` 等为主 epic、TP 标 HINT）→ 必读本概览
2. **按 story 内容** 识别涉及的子类（如"购买成功飘字"→ 涉及 B 资源飘字 + E Toast + D 模态弹窗）
3. **按需加载** 对应子模板
4. **强制交叉参考** [`HINT/O_boundary.md`](./HINT/O_boundary.md) 防止误标 HINT 标签（实际是 UI `F.GUIDE_HINT` 或 BIZ）

## 5. 边界总览（与 `.cursor/MODULES.md` §4.11 一致）

| 归 HINT                          | 不归 HINT（归其他模块）              |
| --------------------------------- | ---------------------------------- |
| 玩家可见的临时反馈、提示内容、文案 | 通知的底层 API、SDK、框架能力 → AUX |
| 红点/飘字/Toast/弹窗文案/触发逻辑 | 业务流程中"什么条件下触发" → BIZ   |
| 错误提示文案分级样式             | 错误码体系、错误处理业务逻辑 → BIZ |
| 活动弹窗的 UI 表现（弹窗本身）| 活动状态机、活动业务规则 → BIZ     |
| 限时倒计时浮窗的展示             | 倒计时定时器、定时任务调度 → BIZ   |
| 合规弹窗内容（防沉迷文案）| 合规校验、风控逻辑 → SPECIAL       |
| 离线补偿弹窗的展示               | 离线补偿数据生成、回档补偿业务 → BIZ |
| 引导气泡、遮罩指引（**触发弹出一次性**）| 引导底层框架（步骤管理器） → AUX   |
| 飘字/Toast 多队列管理            | 通知底层队列框架 → AUX              |
| 承载样式（红点图标、Toast UI）| 提示内容本身（飘字文案、状态码）→ HINT |
