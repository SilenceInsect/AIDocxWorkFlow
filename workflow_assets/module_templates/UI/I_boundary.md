# I. UI 模块边界区分（判定规则）

> **非测试类型**——本文件是 S5 生成测试点时的**判定参考**，
> 避免误标 UI 标签（实际是 BIZ/HINT/AUX/LINK/LOG/SPECIAL）。
>
> **来源**：`.cursor/MODULES.md` §4.5 I

---

## 1. 判定原则

> 凡是涉及**网络请求、后端数据、状态机、外部系统、付费、协议、埋点、提示内容**的测试点，
> 都**不**应标 `UI` 模块，应归对应下游模块。

### 判定三问

1. **是否调用后端接口？** 是 → 不归 UI
2. **是否改变后端数据/状态？** 是 → 不归 UI
3. **是否依赖外部系统（支付/登录/推送）？** 是 → 不归 UI

## 2. 边界对照表

### vs BIZ（业务）

| 场景 | 归 UI | 归 BIZ |
|------|-------|--------|
| 兑换码输入 | 输入框格式校验（前端正则）| 调接口提交兑换码、扣减道具、落库 |
| 购买按钮 | 按钮渲染、状态切换、点击响应 | 调支付接口、扣款、发货、订单生成 |
| 列表筛选 | 纯前端本地筛选/排序/分页 | 调接口获取列表数据 |
| 弹窗确认 | 弹窗打开/关闭、层级、动画 | 确认逻辑（取消订单、确认收货）|
| 表单提交 | 前端格式校验、必填检查 | 接口提交、落库、返回结果处理 |

### vs HINT（提示内容）

| 场景 | 归 UI（承载样式）| 归 HINT（内容本身）|
|------|------------------|-------------------|
| 红点 | 红点图标位置/大小/闪烁动画 | 红点显示/隐藏的逻辑、是否已读 |
| Toast | Toast 弹窗位置/时长/动画 | Toast 显示什么文字（成功/失败/警告）|
| 飘字 | 飘字轨迹/动画/淡入淡出 | 飘字显示什么数字（+100 金币）|
| 系统弹窗 | 弹窗 UI 样式/层级/关闭 | 弹窗内容（"确定要退出吗"）|
| 倒计时 | 倒计时 UI 位置/刷新策略 | 倒计时逻辑（与服务器时间同步）|

### vs AUX（辅助/网络/资源）

| 场景 | 归 UI | 归 AUX |
|------|-------|--------|
| 网络异常 | 断网提示横幅、占位 UI | 网络检测逻辑、自动重连机制 |
| 资源加载 | 资源加载后的渲染、占位 | 资源下载、缓存、命中率 |
| 加载失败 | 失败占位 UI、重试按钮 | 重试机制（指数退避）|
| 超时 | 超时占位 UI | 超时检测（30s timeout）|

### vs LINK（关联/外部系统）

| 场景 | 归 UI | 归 LINK |
|------|-------|--------|
| 第三方登录 | 登录按钮 UI、loading 态 | 调微信/QQ/Apple 登录、token 同步 |
| 第三方支付 | 支付弹窗 UI、按钮点击 | 调支付宝/微信支付、回调处理 |
| 数据同步 | 同步进度条 UI | 多端数据同步协议、冲突解决 |
| 跨服务调用 | 协议返回后的 UI 渲染 | 协议本身、字段映射 |

### vs LOG（日志/埋点）

| 场景 | 归 UI | 归 LOG |
|------|-------|--------|
| 按钮点击 | 按钮按下的 UI 响应 | 按钮点击事件埋点（`btn_click`）|
| 页面访问 | 页面 UI 渲染 | 页面 PV/UV 埋点 |
| 错误提示 | 错误提示 UI 样式 | 错误日志上报（`error_report`）|
| 动效完成 | 动效流畅度 | 动效开始/结束事件埋点 |

### vs SPECIAL（特殊情境/异常处理）

| 场景 | 归 UI | 归 SPECIAL |
|------|-------|-----------|
| 弱网 | 弱网占位 UI | 弱网检测、降级策略、丢包重传 |
| 高频请求 | 防抖按钮 UI（视觉反馈）| 防抖逻辑、节流策略 |
| 切后台 | 切回前台恢复 UI | 切后台处理（暂停计时、断开连接）|
| 重复包 | 重复请求提示 UI | 幂等性、唯一键去重 |
| 反作弊 | 异常行为警告 UI | 反作弊检测、封号策略 |

### vs CONFIG（配置）

| 场景 | 归 UI | 归 CONFIG |
|------|-------|-----------|
| 按钮文字 | 按钮 UI 渲染 | 按钮文字配置（i18n、运营配置）|
| 弹窗内容 | 弹窗 UI 样式 | 弹窗文案配置 |
| 皮肤 | 皮肤 UI 切换 | 皮肤资源配置表 |
| 数值 | 数值显示 UI | 数值计算公式（汇率、概率）|

## 3. 常见误判案例

### 误判 1：购买按钮 → 错标 UI

**错误**：购买按钮按下了，标 UI 完事
**正确**：
- UI 测"按钮按下响应、状态切换、动画"（归 UI）
- BIZ 测"调支付接口、扣款、发货、订单生成"（归 BIZ）
- 同一个 Story 应有 2 个 TP：一个 UI 标签、一个 BIZ 标签

### 误判 2：网络错误 → 错标 UI

**错误**：看到"网络错误"弹窗，就标 UI
**正确**：
- UI 测"网络错误占位 UI、按钮置灰"（归 UI）
- AUX 测"网络检测、自动重连"（归 AUX）
- SPECIAL 测"弱网降级、丢包重传"（归 SPECIAL）

### 误判 3：Toast 提示 → 错标 UI

**错误**：Toast 显示"购买成功"，就标 UI
**正确**：
- UI 测"Toast 弹窗位置/时长/动画"（归 UI）
- HINT 测"Toast 显示什么内容、什么时机触发"（归 HINT）

### 误判 4：登录弹窗 → 错标 UI

**错误**：登录弹窗按设计有 UI 测试点，但没标 LINK
**正确**：
- UI 测"登录弹窗 UI 样式"（归 UI）
- BIZ 测"登录逻辑、token 存储"（归 BIZ）
- LINK 测"调第三方登录（微信/QQ）"（归 LINK）

### 误判 5：埋点验证 → 错标 UI

**错误**：看到 UI 上的事件，标 UI 测
**正确**：
- UI 测"按钮可点击"（归 UI）
- LOG 测"点击后是否上报 `btn_click` 事件"（归 LOG）

## 4. 判定流程图（S5 prompt 决策树）

```
Story 涉及某个功能点
    │
    ├── 涉及后端接口/数据/状态？
    │     是 → 归 BIZ（同时可能配 UI 测试容器）
    │
    ├── 涉及外部系统（支付/登录/同步）？
    │     是 → 归 LINK（同时可能配 UI 测试容器）
    │
    ├── 涉及提示内容（红点/Toast/飘字）？
    │     是 → 归 HINT（同时测 UI 容器）
    │
    ├── 涉及网络/资源/缓存/工具？
    │     是 → 归 AUX（同时可能配 UI 占位）
    │
    ├── 涉及埋点/日志/审计？
    │     是 → 归 LOG（独立测试）
    │
    ├── 涉及异常处理/弱网/反作弊？
    │     是 → 归 SPECIAL（同时可能配 UI 占位）
    │
    ├── 涉及配置读取/计算/联动？
    │     是 → 归 CONFIG（独立测试）
    │
    └── 纯前端 UI（无上述依赖）？
          是 → 归 UI
```
