# O. BIZ 模块边界区分（判定规则）

> **非测试类型**——本文件是 S5 生成测试点时的**判定参考**，
> 避免误标 BIZ 标签（实际是 UI / CONFIG / AUX / SPECIAL / LOG / LINK / HINT）。
>
> **来源**：用户细化定义 §4「模块边界区分」+ §5「游戏专项补充高频测试点」

---

## 1. 判定原则

> **BIZ = 服务端业务逻辑 + 协议 + 数据流 + 状态机 + DB + 并发 + 定时 + 付费 + 审计**
>
> **不归 BIZ**：纯 UI、配置表字段、通用工具、特殊行为、通用日志、第三方 SDK、提示内容

### 判定三问

1. **是"服务端业务逻辑"还是"通用工具/UI/配置"？** 是后者 → 不归 BIZ
2. **是"业务流程"还是"行为异常（弱网/切后台/反作弊）"？** 是行为异常 → 不归 BIZ
3. **是"业务日志"还是"通用行为日志"？** 是通用日志 → 不归 BIZ

## 2. 边界对照表

### vs UI

| 场景 | 归 BIZ | 归 UI |
|------|--------|-------|
| 购买流程 | 扣款、发货、限购、状态机 | 购买按钮样式、支付弹窗 UI |
| 商城 | 商品列表加载、下单业务 | 商品卡片渲染、列表滑动 |
| 任务 | 任务进度更新、奖励发放 | 任务列表 UI、进度条动画 |
| 战斗 | 伤害计算、暴击、状态机 | 战斗 UI、伤害飘字 |
| 邮件 | 邮件收发、附件发放、状态 | 邮件列表 UI、附件领取按钮 |
| 道具 | 产出消耗、堆叠、过期 | 道具图标、Tooltip |

### vs CONFIG

| 场景 | 归 BIZ | 归 CONFIG |
|------|--------|-----------|
| 活动 | 活动业务逻辑、状态机、奖励 | 活动起止时间配置、奖励配置 |
| 道具 | 道具产出消耗、堆叠上限业务 | 道具表字段、ID 合法性 |
| 商城 | 购买流程、限购、扣款 | 商品表字段、价格配置 |
| 概率 | 抽卡保底业务、累计奖励 | 概率配置、配置总和 ≤ 100% |
| 任务 | 任务进度、奖励发放 | 任务表字段、奖励表字段 |
| 数值 | 伤害公式、加成计算 | 公式配置、数值平衡 |
| 热更 | 业务使用新配置生效 | 配置热更流程 |

### vs AUX

| 场景 | 归 BIZ | 归 AUX |
|------|--------|--------|
| 网络 | 业务数据上行下行、时序 | 网络层断线重连、协议打包 |
| 缓存 | 业务缓存策略（排行榜 5min）| Redis 读写、缓存击穿 |
| 加密 | 业务敏感字段加密、签名 | AES 加密算法、密钥管理 |
| 异常 | 业务异常处理（购买失败补偿）| 全局异常兜底、崩溃捕获 |
| 性能 | 业务性能指标 | 帧率监控、内存监控 |
| 定时器 | 业务定时执行（零点重置）| cron 组件 |
| 异步队列 | 业务异步发奖 | 消息队列 SDK |
| GM | GM 业务（发奖业务）| GM 工具/权限 |
| 日志 | 业务审计日志 | 通用日志 SDK |

### vs SPECIAL

| 场景 | 归 BIZ | 归 SPECIAL |
|------|--------|--------|
| 网络 | 业务上行下行 | 弱网/丢包/切后台 |
| 高频 | 业务处理高频请求 | 高频包行为后果（封号）|
| 协议 | 业务协议字段 | 重复包/伪造包/越权 |
| 安全 | 业务风控（付费风控）| 反作弊检测 |
| 状态 | 业务状态机 | 异常状态（断线/切后台）|
| 边界 | 业务异常处理 | 边界/异常场景 |

### vs LOG

| 场景 | 归 BIZ | 归 LOG |
|------|--------|--------|
| 业务审计 | 资源变动、付费审计、状态变更 | 通用行为日志、埋点 |
| 异常 | 业务异常处理 | 异常日志、崩溃日志 |
| GM | GM 操作审计 | 玩家操作日志 |
| 监控 | 业务指标监控 | 性能监控、用户行为 |

### vs LINK

| 场景 | 归 BIZ | 归 LINK |
|------|--------|--------|
| 支付 | 充值订单流程、回调业务 | 第三方支付 SDK 集成 |
| 跨服 | 跨服业务数据、并发 | 跨服务协议、第三方登录 |
| 同步 | 业务数据同步 | 多端一致性、外部 API |

### vs HINT

| 场景 | 归 BIZ | 归 HINT |
|------|--------|--------|
| 提示触发 | 业务触发提示（购买成功飘字）| 提示内容、文案、状态 |
| 红点 | 业务触发红点（新邮件）| 红点 UI、状态机 |
| Toast | 业务触发 Toast | Toast 样式 |
| 弹窗 | 业务触发弹窗 | 弹窗 UI |

## 3. 常见误判案例

### 误判 1：抽卡 → 错标 BIZ
- **错误**：看到"抽卡"，就标 BIZ
- **正确**：
  - BIZ-A 测"抽卡业务逻辑/保底/累计"（归 A）
  - CONFIG-G 测"抽卡概率配置"（归 CONFIG）
  - BIZ-D 测"卡池状态机"（归 D）
  - BIZ-I 测"抽卡审计日志"（归 I）
  - 同一 Story 应有 4-5 个 TP

### 误判 2：商城购买 → 错标 BIZ
- **错误**：看到"商城"，就标 BIZ 一个
- **正确**：
  - BIZ-A 测"购买扣款发货"（归 A）
  - BIZ-B 测"上行下行数据"（归 B）
  - BIZ-C 测"购买协议"（归 C）
  - BIZ-D 测"订单状态机"（归 D）
  - BIZ-H 测"支付订单/对账"（归 H）
  - UI 测"商城 UI 渲染"（归 UI）

## 4. 判定流程图

```
Story 涉及某个功能点
    │
    ├── 是"服务端业务逻辑"？
    │     是 → 归 BIZ（按子类 A-I 选择）
    │           │
    │           ├── 业务规则/计算 → A
    │           ├── 数据链路/时序 → B
    │           ├── 协议字段/错误码 → C
    │           ├── 状态切换/流转 → D
    │           ├── DB 落库/事务 → E
    │           ├── 多玩家并发 → F
    │           ├── 定时/异步 → G
    │           ├── 付费/订单 → H
    │           └── 审计/追溯 → I
    │
    ├── 是"页面视觉/前端交互"？
    │     是 → 归 UI
    │
    ├── 是"配置表字段/数值依赖/热更"？
    │     是 → 归 CONFIG
    │
    ├── 是"通用工具/网络底层/缓存/加密"？
    │     是 → 归 AUX
    │
    ├── 是"弱网/高频包/反作弊/切后台"？
    │     是 → 归 SPECIAL
    │
    ├── 是"通用行为日志/埋点/监控"？
    │     是 → 归 LOG
    │
    ├── 是"第三方 SDK/跨服务/外部 API"？
    │     是 → 归 LINK
    │
    └── 是"提示内容/红点/Toast/飘字"？
          是 → 归 HINT
```
