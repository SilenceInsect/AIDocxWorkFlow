# E. 跨模块资源互通

> **子类代码**：`CROSS_MODULE_RESOURCE`
> **归属模块**：`LINK`
> **来源**：用户细化定义 §三-5-③「跨模块资源互通约束」（原 v1.1 完全缺失，v1.2 新增大类）
>
> **测什么**：**通用货币、通用道具**在**多个模块/系统**之间的全系统流通规则——商城购买产出、任务产出、兑换消耗、礼包发放、数值互通、跨系统限购共享。
> **不测什么**：单系统内部的资源发放（归 BIZ）、资源加载底层实现（归 AUX）、资源显示 UI（归 UI）。
> **与其他子类的差异**：E 关注"**资源/数值**"在多模块之间的互通约束——A 关注"**业务规则**"在多系统之间的联动、D 关注"**外部平台**"对接、F 关注"**对外数据**"透出。

---

## 1. 典型场景

### 场景 1：通用货币"金币"全系统流通
- 业务背景：金币是通用货币，可在商城、任务、合成、礼包等多个系统使用
- 涉及系统：商城、任务、合成、礼包、邮件
- 触发动作：商城花费 100 金币购买道具
- 验证点：金币从账户扣除、商城发货、任务/合成/礼包的金币余额同步显示

### 场景 2：通用道具"材料 A"全系统流通
- 业务背景：材料 A 是通用道具，可从副本掉落、商城购买、礼包发放、任务奖励等获取
- 涉及系统：副本、商城、礼包、任务、合成
- 触发动作：副本掉落材料 A x 5
- 验证点：背包 +5、合成可使用、礼包/任务也能用同一道具

### 场景 3：商城购买产出
- 业务背景：玩家在商城用通用货币购买道具
- 涉及系统：商城、账户系统、背包系统
- 触发动作：购买道具 X
- 验证点：货币扣除、道具入背包、跨系统资源互通约束生效

### 场景 4：任务产出资源
- 业务背景：玩家完成任务获得奖励
- 涉及系统：任务系统、账户系统、背包系统
- 触发动作：完成任务 X
- 验证点：奖励正确发放、通用资源/通用道具正确入背包

### 场景 5：兑换消耗资源
- 业务背景：玩家在兑换商店用通用货币兑换道具
- 涉及系统：兑换商店、账户系统、背包系统
- 触发动作：兑换道具 Y
- 验证点：通用货币扣除、道具入背包、跨系统资源互通约束生效

### 场景 6：礼包发放资源
- 业务背景：玩家领取邮件礼包
- 涉及系统：邮件系统、账户系统、背包系统
- 触发动作：领取礼包 Z
- 验证点：礼包内容正确发放、跨系统资源互通约束生效

### 场景 7：跨系统限购共享
- 业务背景：玩家在 A 商城和 B 商城都购买同一道具
- 涉及系统：A 商城、B 商城、限购系统
- 触发动作：A 商城买 3 次 + B 商城买 2 次
- 验证点：跨商城总限购 5 次（共享池）、超过上限被拦截

### 场景 8：全服每日购买上限
- 业务背景：全服玩家每日购买某道具总数受限
- 涉及系统：全服限购系统、商城
- 触发动作：玩家 A 买 10 个 + 玩家 B 买 10 个
- 验证点：超过全服每日上限（20 个）时 B 玩家第 2 次买被拦截

### 场景 9：全活动共享积分上限
- 业务背景：活动 A、活动 B 共享"节日积分"上限
- 涉及系统：活动 A、活动 B、共享积分池
- 触发动作：A 活动获得 50 积分 + B 活动获得 50 积分（共享上限 80）
- 验证点：B 活动第 N 次获取时被拦截（总积分超过 80）

### 场景 10：货币单位转换
- 业务背景：通用货币"钻石"在不同场景单位不同（充值钻石、游戏内钻石）
- 涉及系统：充值系统、游戏内货币系统
- 触发动作：充值 30 元获得 300 钻石
- 验证点：单位转换正确、跨系统资源互通约束生效

### 场景 11：跨系统资源赠送
- 业务背景：玩家 A 给玩家 B 赠送道具
- 涉及系统：好友系统、账户系统、背包系统
- 触发动作：玩家 A 赠送道具 X x 5 给 B
- 验证点：A 背包 -5、B 背包 +5、跨系统资源互通约束生效

### 场景 12：通用资源过期清理
- 业务背景：临时道具/限时货币过期清理
- 涉及系统：账户系统、背包系统
- 触发动作：限时道具过期
- 验证点：道具从背包消失、跨系统资源互通约束生效（如果该道具被合成引用，合成需重新检查）

### 场景 13：资源互通数值一致性
- 业务背景：跨系统资源互通时数值必须精确
- 涉及系统：账户系统、背包系统、跨系统互通逻辑
- 触发动作：100 钻石 - 30 钻石 + 50 钻石 = 120 钻石
- 验证点：跨多次操作后数值精确（无浮点误差、无四舍五入）

### 场景 14：跨系统限购共享池
- 业务背景：A 活动、B 活动、C 活动都限购"夏日礼包"每日 1 次
- 涉及系统：A 活动、B 活动、C 活动、共享限购池
- 触发动作：A 买 1 次
- **expected**：A 限购 1/1（满），B 和 C 显示 1/1（共享）
- **notes**：注意"共享 vs 独立"——是全活动共享还是各自独立

### 场景 15：资源产出日志追溯
- 业务背景：跨系统资源互通时所有产出/消耗都可追溯
- 涉及系统：账户系统、LOG 系统
- 触发动作：玩家获得/消耗资源
- 验证点：每笔资源变动都有日志、可追溯来源/去向系统

## 3. 边界陷阱

### 边界 1：vs BIZ 业务
- **混淆点**："商城购买"听起来像 BIZ，但"跨系统数值互通"是 E
- **判定规则**：测"单系统内部业务" → BIZ；测"跨系统资源/数值互通" → E
- **实例**：商城扣款发货逻辑 → BIZ；商城扣款后任务/合成/礼包显示同步 → E

### 边界 2：vs CONFIG 配置
- **混淆点**："资源 ID"听起来像 E，但"资源 ID 字段合法性"是 CONFIG
- **判定规则**：测"资源互通规则" → E；测"资源 ID 字段合法性/范围" → CONFIG
- **实例**：资源在多系统间流通的数值规则 → E；资源 ID 字段定义（非负/不重复）→ CONFIG

### 边界 3：vs AUX 底层
- **混淆点**："跨系统数据共享"听起来像 E，但底层数据库是 AUX
- **判定规则**：测"业务互通规则" → E；测"底层能力（数据库/缓存）" → AUX
- **实例**：跨系统限购共享池业务规则 → E；共享池的 Redis 实现 → AUX

### 边界 4：vs LOG 日志
- **混淆点**："资源变动"听起来像 E，但"资源变动日志"是 LOG
- **判定规则**：测"业务互通规则" → E；测"日志/埋点规范" → LOG
- **实例**：跨系统资源数值互通 → E；资源变动的埋点字段规范 → LOG

## 4. 验证证据

### 视觉证据
- 通用货币/道具在多系统（商城/任务/合成/礼包）余额同步显示截图
- 跨系统限购共享池拦截截图（A 买后 B 显示限购满）
- 全服每日购买上限拦截截图

### 日志证据
- `<LOG_CROSS_MODULE_RESOURCE>`：跨系统资源变动，source_system、target_system、amount
- `<LOG_LIMIT_SHARED_POOL>`：跨系统限购共享池变化
- `<LOG_GLOBAL_LIMIT>`：全服限购

### 数据证据
- 数据库 `player_resource` 表：玩家资源账户（单点权威）
- 数据库 `limit_shared_pool` 表：跨系统限购共享池
- 数据库 `global_limit` 表：全服每日购买上限
- 数据库 `resource_flow_log` 表：资源流转日志

### 性能证据
- 跨系统资源同步延迟 < 1s
- 限购共享池并发安全（10 个玩家同时买同一道具）
- 数值计算精确（100 次加减后无误差）
