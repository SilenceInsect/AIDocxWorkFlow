# A. 内部业务关联

> **子类代码**：`INTERNAL_BIZ_LINKAGE`
> **归属模块**：`LINK`
> **来源**：用户细化定义 §三-1「内部业务关联」（原"业务逻辑关联——复开、业务相交、上下游"扩展）
>
> **测什么**：游戏内**多个业务系统**之间的联动规则、依赖关系、数据流转约束；活动复开/重置时的隔离规则；多活动并行的共享/互斥逻辑。
> **不测什么**：单系统独立业务流程（归 BIZ）、底层网络/缓存实现（归 AUX）、异常场景下的安全拦截（归 SPECIAL）。
> **与其他子类的差异**：A 关注"**系统之间**"的业务互通规则——B 关注"**服务之间**"的数据同步、C 关注"**多端之间**"的状态对齐、D 关注"**外部平台之间**"的对接。

---

## 1. 典型场景

### 场景 1：任务解锁商城
- 业务背景：玩家完成主线任务"X 章 Y 节"后解锁商城购买权限
- 涉及系统：任务系统、商城系统、玩家权限系统
- 触发动作：完成解锁任务 + 打开商城
- 验证点：商城商品列表从"未解锁"灰态切换为"可购买"，权限与任务进度同步

### 场景 2：副本通关解锁养成
- 业务背景：通关"困难副本 N"解锁角色养成"X 阶 Y 级"
- 涉及系统：副本系统、角色养成系统、成就系统
- 触发动作：完成副本 + 进入养成界面
- 验证点：养成等级上限提升、副本通关记录影响养成入口

### 场景 3：充值解锁累充奖励
- 业务背景：单笔充值达到 X 元解锁累充活动某一档奖励
- 涉及系统：充值系统、累充活动、邮件系统
- 触发动作：完成充值 + 累充进度刷新 + 邮件发奖
- 验证点：累充进度按金额累加、达到档位自动发邮件、邮件状态机正确

### 场景 4：活动前置条件校验
- 业务背景：限时活动"夏日祭典"需玩家等级 ≥ 40 + 完成前置任务"X"
- 涉及系统：活动系统、任务系统、玩家等级系统
- 触发动作：尝试进入活动
- 验证点：未满足前置时拦截提示、满足后正常进入

### 场景 5：限时活动重复开启
- 业务背景：每周可重新开启的"周末副本"活动
- 涉及系统：活动系统、背包系统、奖励系统
- 触发动作：第二次开启活动
- 验证点：历史进度重置、限购次数清零、阶段性奖励隔离、**新一周数据不污染上周**

### 场景 6：多活动并行共享积分
- 业务背景："春日活动"+"夏日活动"同时开启，共享"节日积分"通用货币
- 涉及系统：活动 A、活动 B、积分系统
- 触发动作：在任一活动中获取/消耗积分
- 验证点：两个活动积分余额实时同步、跨活动消耗正确扣除

### 场景 7：多活动互斥开启
- 业务背景："赛季 A"与"赛季 B"互斥，不能同时参与
- 涉及系统：活动系统、玩家参与状态系统
- 触发动作：尝试开启第二个赛季
- 验证点：提示"已参与其他赛季"、互斥资源扣除互锁

### 场景 8：赛季数据归档不污染
- 业务背景：第 1 赛季结束后开启第 2 赛季
- 涉及系统：赛季系统、排行榜系统、奖励系统
- 触发动作：第 2 赛季开启
- 验证点：第 1 赛季数据归档（不删除，可查）、第 2 赛季进度从零开始、排行榜分赛季展示

### 场景 9：背包道具同步至合成
- 业务背景：合成系统读取背包道具数量判定合成可行性
- 涉及系统：背包系统、合成系统、配置系统
- 触发动作：消耗道具 + 合成产出
- 验证点：背包与合成数据实时一致、合成后背包正确扣减

### 场景 10：装备同步至战力
- 业务背景：玩家更换装备后战力实时刷新
- 涉及系统：装备系统、战力系统、排行榜系统
- 触发动作：穿戴/卸下装备
- 验证点：战力值随装备变化、排行榜同步刷新、跨系统数据联动约束生效

### 场景 11：充值同步累充进度
- 业务背景：玩家充值后"累充奖励"进度条实时刷新
- 涉及系统：充值系统、累充活动系统、邮件系统
- 触发动作：完成一笔充值
- 验证点：累充进度按金额累加、跨档位时自动发奖、邮件状态机正确

### 场景 12：角色等级同步全系统解锁
- 业务背景：玩家升级到 Lv.50 后开放"竞技场"+"公会"+"结婚"等多个系统
- 涉及系统：角色系统、多个子系统（竞技场/公会/结婚）
- 触发动作：升级到 Lv.50
- 验证点：所有解锁入口同步出现、入口显示解锁动画、配置项统一变更

## 2. 种子测试点（TP 模板）

### TP-001（INTERNAL_BIZ_LINKAGE）：任务完成解锁商城
- **scenario**：场景 1
- **module**：`INTERNAL_BIZ_LINKAGE`
- **precondition**：玩家处于"主线任务 X 章节 Y 节"，商城商品列表为"未解锁"灰态
- **test_data**：完成任务 Y 节最后一环
- **expected**：商城商品列表实时切换为"可购买"、权限与任务进度同步刷新、UI 反馈解锁动效
- **notes**：注意"任务完成"到"商城解锁"是异步推送还是实时拉取——异步推送有 1-3s 延迟

### TP-002（INTERNAL_BIZ_LINKAGE）：副本通关解锁养成上限
- **scenario**：场景 2
- **module**：`INTERNAL_BIZ_LINKAGE`
- **precondition**：玩家养成等级上限为 Lv.5、副本"困难 X"未通关
- **test_data**：通关副本"困难 X"
- **expected**：养成等级上限提升至 Lv.10、副本通关记录写入、养成入口允许突破原上限
- **notes**：注意"养成"是 BIZ 系统，但"通关→上限提升"这一联动规则归 A

### TP-003（INTERNAL_BIZ_LINKAGE）：单笔充值解锁累充档位

- **module**：`INTERNAL_BIZ_LINKAGE`
- **precondition**：累充档位 100/500/1000，累充进度 480
- **test_data**：单笔充值 30 元（实际累加 30）
- **expected**：累充进度变为 510、跨越 500 档位自动发邮件、邮件含 500 档奖励
- **notes**：注意"充值"是 BIZ（业务），但"充值→累充进度→邮件发奖"这一跨系统联动归 A

### TP-004（INTERNAL_BIZ_LINKAGE）：活动前置条件未满足拦截

- **test_data**：点击活动入口
- **expected**：提示"等级不足，需 ≥ Lv.40"或"未完成前置任务 X"
- **notes**：注意"拦截逻辑"是 BIZ 业务，但"前置条件的多系统校验"（等级系统 + 任务系统）归 A

### TP-005（INTERNAL_BIZ_LINKAGE）：活动多次开启历史进度重置

## 3. 边界陷阱

### 边界 1：vs BIZ 业务
- **混淆点**："任务完成"是 BIZ，"任务完成→商城解锁"是 A
- **判定规则**：测"单系统内部的业务规则" → BIZ；测"跨系统的业务联动规则" → A
- **实例**：商城内部购买扣款逻辑 → BIZ；任务完成触发商城解锁 → A

### 边界 2：vs AUX 底层
- **混淆点**："活动数据"听起来像 A，但"活动数据存储到数据库"是 BIZ
- **判定规则**：测"业务联动规则" → A；测"底层能力（存储/网络/缓存）" → AUX
- **实例**：活动数据同步刷新规则 → A；活动数据缓存到 Redis 的淘汰策略 → AUX

### 边界 3：vs SPECIAL 异常
- **混淆点**："活动重置时数据隔离"是 A，"活动重置时防止刷数据"是 SPECIAL
- **判定规则**：测"正常业务规则" → A；测"异常场景的安全拦截" → SPECIAL
- **实例**：活动进度重置规则 → A；活动重置时利用漏洞刷奖励 → SPECIAL

### 边界 4：vs C 多端一致性
- **混淆点**："玩家升级→多系统解锁"是 A，"玩家 A 端升级后 B 端同步显示"是 C
- **判定规则**：测"服务端业务规则" → A；测"多端之间的数据同步对齐" → C
- **实例**：服务器侧解锁竞技场入口 → A；PC 端解锁后移动端也看到入口 → C

## 4. 验证证据

### 视觉证据
- 商城解锁前后对比截图（灰态 vs 可购买）
- 活动重置前后背包/限购次数截图
- 多个系统同步解锁的入口截图（红点/新入口）

### 日志证据
- `<LOG_UNLOCK_SHOP>`：任务 X 完成后商城解锁
- `<LOG_ACTIVITY_RESET>`：活动 ID X 进度重置
- `<LOG_CROSS_SYSTEM_SYNC>`：跨系统数据同步推送
- `<LOG_AWARD_MAIL_SENT>`：累充档位发邮件

### 数据证据
- 数据库 `player_unlock` 表：玩家 X 解锁商城 = 1
- 数据库 `activity_progress` 表：活动 X 玩家 Y 进度 0/5
- 数据库 `shared_pool` 表：跨活动共享积分 100
- 数据库 `season_archive` 表：第 1 赛季数据已归档

### 性能证据
- 跨系统同步推送延迟 < 3s（推送 → 接收 → 落库）
- 多系统同步解锁并发数（10 个系统同时解锁）< 5s 完成
- 活动重置时数据清理 < 1s（10 万玩家）
