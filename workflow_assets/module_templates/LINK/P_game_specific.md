# P. LINK 模块游戏项目补充

> **仅游戏项目**——非游戏项目可忽略本文件。
>
> **来源**：用户细化定义 + 游戏项目特殊场景（本地化/活动/礼包/资源/防沉迷/赛季/跨服/渠道）
>
> **作用**：S5 生成 LINK 模块测试点时，对**游戏项目特有**的互通场景做额外检查项。

---

## 1. 典型场景（5 个游戏子领域）

### 场景 1：多区服本地化
- 业务背景：游戏有多个区服（国服/亚服/欧美服），每个区服本地化不同
- 涉及系统：本地化系统、多区服
- 触发动作：玩家在亚服登录
- 验证点：本地化文案/货币/活动按区服生效、跨区服数据不互通

### 场景 2：多区服渠道差异化
- 业务背景：游戏在不同区服接入不同渠道（iOS/Android/H5/小程序）
- 涉及系统：渠道系统、多区服
- 触发动作：玩家在 iOS 渠道登录
- 验证点：渠道专属功能/活动/计费规则按区服+渠道生效

### 场景 3：限时活动跨区服
- 业务背景：限时活动在多个区服同步开启/不同步
- 涉及系统：活动系统、多区服
- 触发动作：活动开启
- 验证点：跨区服活动开启时间/规则/奖励对齐或差异

### 场景 4：礼包跨区服发放
- 业务背景：CDK 礼包跨区服发放（玩家用 A 服 CDK 在 B 服兑换）
- 涉及系统：CDK 系统、多区服
- 触发动作：玩家在 B 服输入 A 服 CDK
- 验证点：CDK 跨区服规则（允许/不允许/限频次）

### 场景 5：资源跨区服交易
- 业务背景：跨区服资源交易（A 服道具 → B 服）
- 涉及系统：交易系统、多区服
- 触发动作：跨区服交易
- 验证点：跨区服交易规则（允许/不允许/汇率/限额）

### 场景 6：赛季跨服/跨区服
- 业务背景：赛季排名跨服/跨区服
- 涉及系统：赛季系统、跨服
- 触发动作：玩家在赛季中
- 验证点：赛季排名跨服/跨区服对齐、跨赛季数据归档

### 场景 7：跨服公会战
- 业务背景：公会战跨服（A 服公会 vs B 服公会）
- 涉及系统：公会系统、跨服
- 触发动作：公会战开始
- 验证点：跨服对战数据同步、奖励跨服发放

### 场景 8：跨服交易行
- 业务背景：跨服交易行（A 服玩家可买 B 服玩家上架的道具）
- 涉及系统：交易行系统、跨服
- 触发动作：A 服玩家买 B 服玩家上架的道具
- 验证点：跨服撮合、跨服扣款发货

### 场景 9：渠道专属活动
- 业务背景：iOS 渠道专属活动（iOS 玩家独享）
- 涉及系统：渠道系统、活动系统
- 触发动作：iOS 玩家登录
- 验证点：iOS 玩家看到活动、Android 玩家看不到

### 场景 10：跨区服玩家迁移
- 业务背景：玩家从 A 区服迁移到 B 区服（如数据合并/跨区服合服）
- 涉及系统：玩家系统、跨区服
- 触发动作：玩家迁移
- 验证点：玩家数据完整迁移、迁移后跨区服可继续玩

### 场景 11：灰度服新功能互通
- 业务背景：灰度服新功能在白名单玩家中互通
- 涉及系统：灰度系统、跨服
- 触发动作：白名单玩家进入灰度服
- 验证点：白名单玩家看到新功能、与正式服数据隔离但白名单维度互通

### 场景 12：跨区服活动奖励发放
- 业务背景：跨区服活动完成后奖励跨区服发放
- 涉及系统：活动系统、跨区服、邮件系统
- 触发动作：跨区服活动完成
- 验证点：奖励按区服发放、跨区服不可互用

## 2. 种子测试点（TP 模板）

### TP-001（LINK-A 本地化）：多区服本地化生效
- **module**：`INTERNAL_BIZ_LINKAGE`（A.内部业务关联 — 多系统联动规则）
- **precondition**：游戏有国服/亚服/欧美服
- **test_data**：玩家在亚服登录
- **expected**：亚服本地化文案/货币/活动按区服生效
- **notes**：注意"按区服维度"——不是全局，是按区服独立配置

### TP-002（LINK-D 渠道）：渠道差异化活动生效
- **module**：`EXTERNAL_THIRD_PARTY`（D.外部第三方 — 渠道对接）
- **precondition**：iOS 渠道专属活动"iOS 玩家独享"
- **test_data**：iOS 玩家登录 + Android 玩家登录
- **expected**：iOS 玩家看到活动、Android 玩家看不到
- **notes**：注意"渠道 + 区服"双维度——iOS 国服 vs iOS 亚服规则可能不同

### TP-003（LINK-B 跨服）：限时活动跨区服同步
- **module**：`CROSS_SERVER_SYNC`（B.跨服务/分布式 — 跨服同步）
- **precondition**：限时活动"夏日祭典"在国服+亚服同步开启
- **test_data**：活动开启时间
- **expected**：国服 + 亚服同时开启、跨区服活动规则/奖励对齐
- **notes**：注意"同步 vs 异步"——同步开启（如全球活动）、异步开启（如节日活动按时区）

### TP-004（LINK-D 跨区服）：CDK 跨区服规则
- **module**：`EXTERNAL_THIRD_PARTY`（D.外部第三方 — 外部对接）
- **precondition**：玩家在 A 服获得 CDK "X123"
- **test_data**：玩家在 B 服输入 CDK
- **expected**：按配置规则——允许/不允许/限频次（如每周限 1 次）
- **notes**：注意"跨区服 vs 跨服"——跨区服（A 国服 vs B 亚服）、跨服（A 服 1 vs A 服 2）

### TP-005（LINK-E 资源）：跨区服资源交易规则
- **module**：`CROSS_MODULE_RESOURCE`（E.跨模块资源互通 — 资源/数值）
- **precondition**：A 服玩家有道具 X x 10、B 服玩家想买
- **test_data**：跨区服交易
- **expected**：按配置规则——允许/不允许/汇率（A 服 1 金币 = B 服 X 金币）/限额
- **notes**：注意"汇率"——不同区服货币单位可能不同，需汇率换算

### TP-006（LINK-B 赛季）：赛季排名跨服
- **module**：`CROSS_SERVER_SYNC`（B.跨服务/分布式 — 跨服同步）
- **precondition**：第 1 赛季跨服进行中
- **test_data**：玩家 A 服 + 玩家 B 服 都在赛季中
- **expected**：跨服赛季排名正确、跨赛季数据归档
- **notes**：注意"跨服 vs 跨区服"——跨服（同一区服不同服）、跨区服（不同区服）

### TP-007（LINK-A 跨服公会）：跨服公会战
- **module**：`INTERNAL_BIZ_LINKAGE`（A.内部业务关联 — 跨系统联动）
- **precondition**：A 服公会 vs B 服公会 进入跨服公会战
- **test_data**：跨服公会战开始
- **expected**：跨服对战数据同步、奖励跨服发放
- **notes**：注意"公会"本身是 BIZ，但"跨服公会战"是 A 联动

### TP-008（LINK-B 跨服交易行）：跨服撮合
- **module**：`CROSS_SERVER_SYNC`（B.跨服务/分布式 — 跨服同步）
- **precondition**：A 服玩家上架道具、B 服玩家想买
- **test_data**：B 服玩家购买
- **expected**：跨服撮合、跨服扣款发货、撮合服宕机时回滚
- **notes**：注意"撮合服"——是 B（跨服服务），但"交易行"UI 是 UI

### TP-009（LINK-D 渠道专属）：iOS 渠道专属活动
- **module**：`EXTERNAL_THIRD_PARTY`（D.外部第三方 — 渠道对接）
- **precondition**：iOS 渠道专属活动"iOS 玩家独享"
- **test_data**：iOS 玩家 + Android 玩家登录
- **expected**：iOS 看到、Android 看不到、按渠道维度生效
- **notes**：注意"渠道识别"——是 SDK 上报还是平台标识？

### TP-010（LINK-B 跨区服迁移）：玩家跨区服迁移
- **module**：`CROSS_SERVER_SYNC`（B.跨服务/分布式 — 跨服同步）
- **precondition**：玩家在 A 区服等级 30、VIP 5
- **test_data**：玩家迁移到 B 区服
- **expected**：玩家数据完整迁移、迁移后可继续玩
- **notes**：注意"迁移 vs 复制"——是数据迁移（原区服无数据）还是复制（双区服都有）

### TP-011（LINK-B 灰度）：白名单灰度新功能互通
- **module**：`CROSS_SERVER_SYNC`（B.跨服务/分布式 — 灰度/环境隔离）
- **precondition**：白名单玩家 A 在灰度服、非白名单 B 在正式服
- **test_data**：运营更新灰度配置（新功能开关=1）
- **expected**：A 看到新功能、B 看不到、按白名单维度生效
- **notes**：注意"白名单生效"——不是"环境生效"

### TP-012（LINK-A 跨区服活动奖励）：跨区服活动奖励发放
- **module**：`INTERNAL_BIZ_LINKAGE`（A.内部业务关联 — 跨系统联动）
- **precondition**：跨区服活动"夏日祭典"完成
- **test_data**：国服 + 亚服玩家都参与活动
- **expected**：奖励按区服发放、跨区服不可互用（国服奖励不能在亚服用）
- **notes**：注意"奖励可移植性"——某些游戏允许跨区服用、某些不允许

### TP-013（LINK-E 赛季积分）：跨赛季积分互通
- **module**：`CROSS_MODULE_RESOURCE`（E.跨模块资源互通 — 资源/数值）
- **precondition**：第 1 赛季积分 1000、第 2 赛季开始
- **test_data**：第 2 赛季开启
- **expected**：第 1 赛季积分归档（可查）、第 2 赛季从 0 开始、跨赛季积分不互通
- **notes**：注意"积分资源"——E 资源互通，但"赛季状态机"是 BIZ

### TP-014（LINK-C 多端）：跨区服多端登录
- **module**：`MULTI_CLIENT_SYNC`（C.多端一致性 — 端到端同步）
- **precondition**：玩家在 A 区服 PC 端 + 移动端双端登录
- **test_data**：移动端跨区服到 B 区服（如果支持）
- **expected**：跨区服后多端状态重新对齐、按区服独立
- **notes**：注意"跨区服 + 多端"双重维度

### TP-015（LINK-F 对外透出）：区服数据对外透出
- **module**：`OUTBOUND_DATA`（F.对外数据透出 — 对外数据）
- **precondition**：国服 + 亚服 + 欧美服 玩家数据需对外透出
- **test_data**：BI 系统拉取各区服数据
- **expected**：按区服维度返回、数据对齐规范、脱敏生效
- **notes**：注意"区服作为字段"——对外数据需含区服标识

## 3. 边界陷阱

### 边界 1：vs CONFIG 配置
- **混淆点**："多区服本地化"听起来像 CONFIG（配置），但"跨区服业务规则"是 LINK
- **判定规则**：测"配置定义" → CONFIG；测"跨区服业务规则" → LINK
- **实例**：本地化文案字段定义 → CONFIG；本地化按区服生效 → LINK A

### 边界 2：vs BIZ 业务
- **混淆点**："公会战"听起来像 BIZ，但"跨服公会战"是 LINK
- **判定规则**：测"单区服单服内业务" → BIZ；测"跨服/跨区服业务" → LINK
- **实例**：公会战内部玩法 → BIZ；跨服公会战同步 → LINK A

### 边界 3：vs SPECIAL 异常
- **混淆点**："跨区服迁移"听起来像 LINK，但"迁移时数据安全"是 SPECIAL
- **判定规则**：测"正常跨区服规则" → LINK；测"异常安全拦截" → SPECIAL
- **实例**：跨区服数据迁移业务 → LINK；迁移时数据防丢失/防篡改 → SPECIAL

### 边界 4：vs LOG 日志
- **混淆点**："跨区服日志"听起来像 LOG，但"跨区服业务"是 LINK
- **判定规则**：测"业务规则" → LINK；测"日志规范" → LOG
- **实例**：跨区服活动奖励发放 → LINK；跨区服操作日志 → LOG
