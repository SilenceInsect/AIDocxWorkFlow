# E. 配置解析与加载

> **子类代码**：`PARSE_LOAD`
> **归属模块**：`CONFIG`
> **来源**：用户细化定义 §5(1)「配置解析与加载逻辑」
>
> **测什么**：客户端/服务端双端解析一致性、超大配置表性能、复杂嵌套结构解析、非法配置容错。
> **不测什么**：单字段合法性（归 A）、跨表 ID 引用（归 C）、热更（归 D）、版本兼容（归 F）。
> **与其他子类的差异**：E 关注"解析层"——A 关注"配置层"，D 关注"运行时变更"。

---

## 1. 典型场景

### 场景 1：客户端/服务端双端解析
- 业务背景：同一份配置文件
- 涉及端：客户端 + 服务端
- 触发动作：双端同时加载配置
- 验证点：解析结果完全一致

### 场景 2：超大配置表加载
- 业务背景：万行配置
- 涉及表：10000+ 行的道具表
- 触发动作：客户端/服务端加载
- 验证点：加载耗时 < 5s、内存 < 100MB

### 场景 3：嵌套结构解析
- 业务背景：多奖励/多条件/多阶段
- 涉及字段：嵌套 JSON
- 触发动作：解析深层结构
- 验证点：无数据丢失

### 场景 4：非法配置容错
- 业务背景：错误配置上线
- 涉及字段：必填字段缺失
- 触发动作：服务端启动
- 验证点：不宕机、有日志告警

### 场景 5：客户端启动加载
- 业务背景：客户端冷启动
- 触发动作：客户端首次启动
- 验证点：所有必需配置加载完成

### 场景 6：服务端启动加载
- 业务背景：服务端冷启动
- 触发动作：服务端首次启动
- 验证点：所有必需配置加载完成

## 3. 边界陷阱

### 边界 1：vs A. 字段合法性
- **混淆点**：「字段"类型"不匹配"」——A 测配置层、E 测解析层
- **判定规则**：测"导出工具拒绝非法字段" → A；测"运行时解析兼容" → E
- **实例**：字段类型 int 录入 string → A-001；运行时兼容 → E-008

### 边界 2：vs C. 跨表依赖
- **混淆点**：「解析"跨表"」——C 测引用存在性、E 测加载性能
- **判定规则**：测"任务表引用道具表 ID 存在" → C；测"加载跨表引用的耗时" → E
- **实例**：任务表引用道具表 → C-001；5 层引用链加载 → E-005

### 边界 3：vs D. 热更新
- **混淆点**：「热更"加载"」——D 测热更流程、E 测首次加载
- **判定规则**：测"运行时热更切换" → D；测"冷启动首次加载" → E
- **实例**：热更生效 → D-001；首启加载 → E-010

### 边界 4：vs I. 服务端专属
- **混淆点**：「服务端"配置"」——I 测服务端业务配置、E 测通用解析
- **判定规则**：测"服务端全局常量生效" → I；测"服务端配置解析性能" → E
- **实例**：开服时间配置 → I-001；服务端配置解析耗时 → E-011

- 解析开始/结束日志

- 解析结果 Hash 对比
- 内存快照

- 加载耗时 < 5s
- 内存占用 < 100MB
- CPU 占用 < 30%
