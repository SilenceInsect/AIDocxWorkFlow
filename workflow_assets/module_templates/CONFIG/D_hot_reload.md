# D. 配置热更新

> **子类代码**：`RELOAD_4_MODE`
> **归属模块**：`CONFIG`
> **来源**：用户细化定义 §4「配置热更新」（大幅补充游戏专项场景）
>
> **测什么**：单表/多表/增量/全量热更、热更前后状态、热更异常兜底、灰度热更。
> **不测什么**：单字段合法性（归 A）、跨表 ID 引用（归 C）、版本兼容（归 F）。
> **与其他子类的差异**：D 关注"运行时配置变更"——A/B/C 关注"配置本身"。

---

## 1. 典型场景

### 场景 1：单表热更
- 业务背景：道具表热更
- 触发动作：修改道具 100 攻击力从 100 → 200
- 验证点：客户端/服务端立即生效

### 场景 2：多表批量热更
- 业务背景：活动期间同时热更 5 张表
- 触发动作：道具表 + 礼包表 + 任务表 + 商城表 + 邮件表 同时热更
- 验证点：5 张表全部生效、无遗漏

### 场景 3：增量热更
- 业务背景：只更新了 1 个道具
- 触发动作：热更包只含 item_id=100 的一行
- 验证点：只更新该行、其他行不重载

### 场景 4：全量配置重载
- 业务背景：服务端启动时全量加载
- 触发动作：触发全量重载
- 验证点：所有表全部重载、耗时 < 30s

### 场景 5：客户端热更
- 业务背景：游戏内 UI 显示
- 触发动作：道具图标热更
- 验证点：客户端 UI 立即刷新

### 场景 6：服务端热更
- 业务背景：服务端逻辑
- 触发动作：掉率配置热更
- 验证点：服务端逻辑立即生效

### 场景 7：玩家在线热更
- 业务背景：玩家正在游戏中
- 触发动作：热更时玩家已打开界面/未打开界面
- 验证点：已打开实时刷新、未打开下次读取新

### 场景 8：离线热更
- 业务背景：玩家离线后重启
- 触发动作：重启客户端/服务端
- 验证点：配置生效正常

### 场景 9：热更包缺失
- 业务背景：热更包下载失败
- 触发动作：网络中断/包损坏
- 验证点：自动回滚至上一版稳定配置

### 场景 10：热更冲突
- 业务背景：新旧配置字段冲突
- 触发动作：热更后字段类型不匹配
- 验证点：程序兼容兜底逻辑生效

### 场景 11：灰度热更
- 业务背景：分批次热更
- 触发动作：先热更 10% 玩家，验证后再全量
- 验证点：分服/分渠道/分批生效校验

## 3. 边界陷阱

### 边界 1：vs A. 字段合法性
- **混淆点**：「热更"后字段"」——A 测字段、D 测热更
- **判定规则**：测"字段本身合法" → A；测"热更前后字段变更生效" → D
- **实例**：新字段类型 int → A-001；热更后字段生效 → D-001

### 边界 2：vs F. 版本兼容
- **混淆点**：「新旧版本"配置"」——F 测版本兼容、D 测运行时热更
- **判定规则**：测"新/旧客户端读新/旧配置" → F；测"运行时热更切换" → D
- **实例**：旧客户端读新配置 → F-001；运行时切换配置 → D-001

### 边界 3：vs I. 服务端专属
- **混淆点**：「服务端"热更"」——I 测服务端配置、D 测热更流程
- **判定规则**：测"服务端全局常量生效" → I；测"服务端热更流程" → D
- **实例**：开服时间配置生效 → I-001；服务端配置热更 → D-007

### 边界 4：vs LOG
- **混淆点**：「热更"日志"」——D 测热更本身、LOG 测埋点
- **判定规则**：测"热更是否生效/不崩溃" → D；测"热更事件埋点上报" → LOG
- **实例**：热更无闪退 → D-002；上报 `config_hot_reload` 事件 → LOG

## 4. 验证证据

### 视觉证据
- 热更前后数值对比截图
- 热更过程录屏

### 日志证据
- 热更开始/结束日志
- 热更回滚日志
- 热更冲突告警日志

### 数据证据
- 配置文件版本号（hot_reload_version）
- 热更包 MD5
- 回滚记录

### 性能证据
- 热更耗时 < 5s
- 热更过程 CPU 占用 < 20%
- 内存无泄漏
