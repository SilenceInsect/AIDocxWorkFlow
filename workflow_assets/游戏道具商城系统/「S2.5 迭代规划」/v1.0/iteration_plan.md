## 2. 迭代目标

1. **核心支付闭环**：游戏币 + 人民币支付 + 回调幂等（第 1-2 轮）
2. **商城展示**：首页 + 详情 + 搜索 + 分类（第 1-2 轮）
3. **VIP + 促销**：折扣体系 + 满减 + 限时（第 3-4 轮）
4. **特殊场景**：弱网 + 高并发 + 风控 + 金额篡改（第 4-5 轮）
5. **监控/审计**：埋点 + 流水 + 风控日志（第 5-6 轮）
6. **灰度上线**：5% → 20% → 100%（第 6 轮）

## 5. 风险点

- **R1**: 第三方支付回调延迟（外部依赖）→ 异步对账 + 客服介入兜底
- **R2**: 10000 并发在 Sprint 5 验证，可能压测机器不足 → 提前 1 周预约
- **R3**: VIP 折扣与限时叠加规则争议（业务方）→ 已确认"取较低价"
- **R4**: 金额篡改攻击频繁 → Sprint 5 重点压测

## 6. 准入准出

- 每个 Sprint 结束：S6 用例覆盖 ≥ 80% + S7 审查 PASS
- 整体准出门禁：所有 P0 用例 100% 通过 + 弱网/高并发/风控三大类 100% 通过
