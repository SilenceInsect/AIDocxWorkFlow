# L. 运营辅助（后台底层工具）

> **子类代码**：`OPS_TOOL`
> **归属模块**：`UTIL`
> **来源**：用户细化定义 §14「运营辅助工具」
>
> **v1.6.1 变更**：
> - 剔除**第三方平台对接**（微信/支付宝/应用商店）→ **LINK 模块**（v1.9 待建）
> - 剔除**运营业务功能**（批量发奖、公告后台业务、活动配置业务）→ **BIZ 模块**（v1.11 待建）
> - 本子模板**只保留运营后台底层工具**（对账统计、灰度白名单工具等）
>
> **测什么**：对账统计工具、灰度白名单底层工具、运营活动数据导出工具、运营查询工具。
> **不测什么**：GM 指令（归 G）、第三方平台（归 LINK）、业务（归 BIZ）、UI（归 UI）、网络（归 B）、公告业务（归 BIZ）。

## 3. 边界陷阱

### 边界 1：vs G. GM 工具
- **混淆点**：「运营"工具"」——G 测 GM 指令、L 测运营底层
- **判定规则**：测"GM 指令（发道具/改等级）" → G；测"对账/灰度底层工具" → L
- **实例**：`gm_add_item` → G；对账统计 → L

### 边界 2：vs BIZ
- **混淆点**：「运营"业务"」——L 测运营工具、BIZ 测运营业务
- **判定规则**：测"对账/灰度底层工具" → L；测"运营业务（发奖/公告/活动配置）" → BIZ
- **实例**：对账工具 → L；批量发奖 → BIZ

### 边界 3：vs CONFIG-H
- **混淆点**：「活动"配置"」——L 测运营工具、CONFIG 测配置
- **判定规则**：测"运营后台 UI 工具" → L；测"配置文件发布" → CONFIG-H
- **实例**：活动配置后台 → L；Excel 导出 → CONFIG-H

### 边界 4：vs LINK
- **混淆点**：「第三方"工具"」——L 测运营底层、LINK 测第三方
- **判定规则**：测"对账工具" → L；测"第三方支付对接" → LINK
- **实例**：对账工具 → L；微信支付 SDK → LINK
