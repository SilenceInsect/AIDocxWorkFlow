> **来源**：用户 2026-07-22 上午反馈——"职责边界"说法不准确，应为"能力边界"，能力范围内的事情都可以做。

## 1. 底座节结构（10 + 1）

### 1.1 为什么是"能力边界"而不是"职责边界"

| 说法 | 含义 | 正确性 |
|------|------|--------|
| **职责边界** | "这件事归我管，那件事不归我管" | ❌ 边界不是职责划的，是模块划的 |
| **能力边界** | "我有没有这个能力，决定我能不能做这件事" | ✅ 能力决定边界 |

> 核心逻辑：我有没有 UI 控件状态建模能力 → 有 → UI 相关的 TP 我都能做 → 不存在"职责边界"，只有"能力范围内都可以做"。

### 1.2 节结构总览

```
身份卡 = 11 节（强制）+ 专家特色节（1+ 节，可选）
       ↓
       节 1-7：身份层（我是谁、资产、目标、标准、价值）
       节 8-13：执行层（哲学、方法、工具、格式、承诺、底线）
       节 14：示例层（实际场景）
       节 15+：特色节（专家差异化）
```

### 1.3 核心原则

1. **能力范围内都可以做**——不存在"我不主动管"的说法，只有"我有没有这个能力"
2. **没有职责边界，只有能力边界**——边界由能力决定，不由职责决定
3. **定量优于定性**——目标、标准、价值都要有量化指标
4. **价值必须可评估**——价值定义和价值评估配套
