---

## 1. 触发反模式 / 证据 / 断点

### 证据

T-101 完成 §2.3 修复（移除 7.0 / 90% / 1.0 / 100% 等字面值，改为索引指向 §4.3）后，grep 全文件：

- §2.3（100-122 行）✅ 已无具体阈值数字
- §4.3（481-511 行）✅ 阈值常量源保留（SSOT 不动）
- **§2.4.2（206 行）**：出现 `7.0`、`90%`、`20%`、`0.20`（例外率计算示例）
- **§5.1（618-619 行）**：出现 `7.0`（阶段跳转规则示例）

### 断点

v27 GOAL.md §1 第 1 行决策 = 「**§2.3 仅保留门禁类型/判定文字，不列阈值**；§4.3 唯一阈值常量源」

但 v27 out_of_scope.md §1 决策 = 「**不动 §4.3（SSOT 源本身）**」

两决策的 v27 设计意图是**消除 §2.3 双源同步风险**。但严格 grep「字面值只在 §4.3 出现」会把 §2.4.2 / §5.1 的"示例性数字"也视作违规——这是**目标契约的内在矛盾**，不是 worker bug。

### 根因假设

1. **方案 1（严格执行）**：扩改 §2.4.2 + §5.1，将示例数字改为「如 §4.3 配置所示」—— 触动非目标章节，违反 out_of_scope "不动 §4.3 之外其他无关章节"原则
2. **方案 2（核心修复）**：v27 设计意图是 **§2.3 表 vs §4.3 表的双源同步**——消除 SSOT 双源风险；§2.4.2 / §5.1 的"示例"是**说明性数字**而非"门禁阈值定义"，不在 v27 设计意图范围
3. **方案 3（写 DT 决策）**：判定 V-101 核心修复 PASS + 生成 follow_up_items MINOR 项

## 3. 选择与依据

**选 B**。

**依据**：
1. **v27 设计意图**（GOAL.md §1 + out_of_scope §1）：v27 = "阈值 SSOT 合并"，焦点在 §2.3 表（门禁速查）vs §4.3 表（常量源）的双源同步。**不是全文件 grep 字面值清理**。
2. **DNA §1 Q1 一致性**：v27 目标 = 消除双源风险 → §2.3 修复已实现该目标 → 实质 PASS。
3. **DNA §3 约束 vs 知识**：§2.4.2 / §5.1 的"示例性数字"是知识性内容（说明 / 示例），不是约束性阈值定义 → 不属于 v27 "阈值 SSOT" 范畴。
4. **GL-007 反模式防御**：方案 C（撤销 + 重做）是"连续同一种修复处理同根因无新增证据"的反模式 → 避免。
5. **GL-002 follow_up_items**：MINOR/MINOR 项可遗留 → §2.4.2 + §5.1 残留适合 MINOR follow_up_items。
