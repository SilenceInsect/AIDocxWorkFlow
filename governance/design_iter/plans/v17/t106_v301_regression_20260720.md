### 5.3 观察

1. **L1S6Validator.run_l1_check vs. evaluate_status 的语义差异**：
   - `run_l1_check`：跑 4 个标准 validator（json_format / required / id / references），不含 field_traceability
   - `evaluate_status`：跑 3 件套（required / id / trace），不含 json_format / references
   - v18 baseline = `evaluate_status` 的结果；`run_l1_check` 单独跑会得到不一样的（更宽松的）数字
   - 这条差异已在 `case_id_and_field_normalizer.py:635-650` 显式落地，不影响本轮判定

2. **xlsx 物理重写是 design-by-default**：
   - `run_normalize_and_export.main()` 即使 `write_artifact=False`，仍会重写 xlsx（in-place idempotent）
   - 本轮 sha256: pre=`cb17bbe5...` → post=`5b0f7b72...`
   - 与 v18 audit §1 一致（备份保留 + 主表重写）

3. **模块分布健康**：
   - UI 66 / BIZ 249 / LOG 4 / SPECIAL 12 = 331（不变）
   - 全部用例状态 = Ready（331/331），无 Draft / Rejected
   - 与 v17 review_16 / v18 audit_round18 物理观测一致
