# TC 内部结构化映射规范

## 1. 问题定义

### 当前问题（v3.01）

测试用例按"1 步 1 TC" 模式生成：

```json
TC-001: {"steps": [{"step_num": 1, "action": "玩家点击商城入口"}]}
TC-002: {"steps": [{"step_num": 2, "action": "系统加载商城首页"}]}
TC-003: {"steps": [{"step_num": 3, "action": "观察道具列表..."}]}
```

**问题**：
- 步骤碎片化，用例不可执行
- 步骤与预期无法对应
- Excel 渲染质量差

### 期望模式（用户要求）

**四层映射关系**：
```
1 用例描述 = N 功能描述
1 功能描述 = N 前置条件
1 前置条件 = N 操作步骤
1 预期结果 = N 操作步骤（步骤-预期对应）
```

## 2. 四层映射模型

| 层级 | 关系 | 说明 |
|------|------|------|
| L1 用例描述 | = S2 obj_name | 需求对象名称，1:1 |
| L2 功能描述 | = S2 fp_name | 功能点名称，1:N（多个测试场景） |
| L3 前置条件 | = S5 TP.preconditions + variant | 测试初始状态，1:N（多种前置） |
| L4 步骤+预期 | = LLM 推理生成 | 操作步骤 + 预期结果，N:1（每前置1条TC） |

## 3. 新 TC Schema

```json
{
  "case_id": "UI-TC-001",
  "module": "UI",
  "用例描述": "商城首页道具列表",

```

## 4. 禁止模式

| 禁止 | 错误示例 | 正确做法 |
|------|---------|---------|
| 1 步 1 TC | TC-001: 玩家点击商城入口 | 合并为1条TC，含3+步骤 |
| 步骤碎裂 | TC-001~TC-005 每条1步 | 1 TC 含连贯步骤序列 |
| 预期缺失 | expected_results: [] | 至少1个预期 |
| 前置为空 | "前置条件": "" | 继承 TP.preconditions |
| 无 step_ref | expected: ["验证通过"] | 显式 step_ref 对应步骤 |

## 5. Excel 渲染规则

### 步骤列渲染

```python
def render_steps(steps):
    """输出格式：每步一行，格式为 "{step_num}. {action}" """
    return "\n".join(f"{s['step_num']}. {s['action']}" for s in steps)
```

### 预期列渲染

```python
def render_expected(expected):
    """输出格式：按 step_ref 对应 "[步骤{step_ref}] {预期}" """
    return "\n".join(f"[步骤{e['step_ref']}] {e['预期']}" for e in expected)
```

### 渲染结果

| 操作步骤 | 预期结果 |
|---------|---------|
| 1. 玩家点击商城入口<br>2. 系统加载商城首页<br>3. 观察道具列表排序 | [步骤1] 商城首页正常打开<br>[步骤2] 道具列表展示前10个道具<br>[步骤3] 每个道具显示完整名称 |

## 6. 自检清单

- [ ] 无步骤碎裂（同一 TC 内 steps 数组连续，无 1 步 1 TC 模式）
- [ ] 预期结果数 ≥ 1
- [ ] 每条预期含显式 step_ref
- [ ] 前置条件非空

> **步骤数要求**：按业务风险灵活调整（典型 2-3 步），无硬性阈值。
