## §2 DNA §10 6 条执行清单

### 10.1 不用指代符

- [ ] 响应中无"这个 / 它 / 上述 / 前者 / 后者"——改用具体名词
- [ ] 示例：`v17 §3.1` 不写"`§3.1`"

### 10.2 说人话

- [ ] 无术语堆砌（拒绝 jargon 堆砌）
- [ ] 无多层嵌套（拒绝 5 层 if）
- [ ] 技术正确 + 人易懂

### 10.3 列执行清单给人工审查

- [ ] 不只写叙述——写"做了什么 + 怎么验证 + 影响范围"
- [ ] 用编号清单（1./2./3.）
- [ ] 每条可独立验证

### 10.4 基于已有实现推理和复用

- [ ] 改之前 grep / Read 看现有文件有什么
- [ ] 不凭空"造"新机制——能复用 `ai_workflow/` 现有函数就用现有函数
- [ ] 不引用未读过的文档——未读过的明说"未核实"

### 10.5 不产出无根据结论

- [ ] 边界说不清就明说"未知 / 未核实 / 待补"
- [ ] 不把"候选规则"说成"已发布硬约束"
- [ ] 不把"CHANGELOG 描述"当作"实际代码行为"

### 10.6 与 §1 Q5 关系

- [ ] §1 是问"有没有做"，§10 是问"做得对不对"
