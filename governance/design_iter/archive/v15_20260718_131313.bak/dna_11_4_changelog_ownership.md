## 改动方案

**文件**: `.cursor/rules/DNA_3Q_CHECK.mdc`
**位置**: §11.1 之后、§11.2 之前
**操作**: StrReplace 插入新段 `### 11.4 产物与变更分离规则`

### 11.4 草稿内容（待用户审）

```markdown
### 11.4 产物与变更分离规则（永久规范）

1. **产品产物**（如 test_points.json / test_cases.json / review_report.md 等）只描述"当前是什么"，不内嵌版本标记、变更时间戳、变更说明
2. **变更记录**统一归口到对应文件：
 - 规范/约束文件（.mdc / SKILL.md / AGENTS.md）的变更 → `CHANGELOG.md`
 - 项目级规范迭代方案 → `governance/design_iter/plans/v{N}.md`
 - 单次需求产物迭代 → `<req_name>/<version>/...` 目录结构（v8 主轴变更后的版本分层）
3. **产品迭代/变更走对应模块**——S5 的迭代走 S5 自己的字段（如 `is_assumed` / `covered` / `skip_reason`）；S6 的迭代走 S6 自己的字段（如 `case_type` / `priority`）；不允许"为了记录变更"在产物里硬塞 `version_note` / `changelog` / `last_modified` 等非业务字段
```
