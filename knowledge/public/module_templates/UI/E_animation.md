# E. 动效与动画

> **子类代码**：`ANIMATION`
> **归属模块**：`UI`
> **来源**：`.cursor/MODULES.md` §4.5 E
>
> **测什么**：页面转场、控件入场退场、按钮反馈、进度条、飘字、闪烁、加载动画的**流畅度与不阻塞**。
> **不测什么**：动画触发后的业务逻辑（归 BIZ）、动画的资源加载（归 D）。
> **与其他子类的差异**：E 关注"过程"——A/B 关注静态状态、D 关注静态展示、E 关注动态过程。

---

## 1. 典型场景

### 场景 1：页面转场
- 业务背景：游戏内页面切换
- 触发动作：从商城页 → 道具详情页
- 验证点：转场动画流畅、不白屏、不卡顿

### 场景 2：控件入场/退场
- 业务背景：弹窗打开/关闭
- 触发动作：点击触发弹窗
- 验证点：弹窗从中心放大/缩小/淡入淡出

### 场景 3：按钮点击反馈
- 业务背景：购买按钮
- 触发动作：点击按钮
- 验证点：按钮按下缩小、释放弹起（按设计）

### 场景 4：进度条
- 业务背景：加载中
- 触发动作：触发加载
- 验证点：进度条平滑前进、不卡顿、不回退

### 场景 5：飘字动画
- 业务背景：获得奖励
- 触发动作：发放奖励
- 验证点：飘字从起点飞向目标、淡入淡出、不卡顿

### 场景 6：闪烁提示
- 业务背景：未读消息
- 触发动作：有新消息
- 验证点：红点闪烁、频率合理

### 场景 7：加载动画
- 业务背景：列表加载
- 触发动作：下拉刷新
- 验证点：spinner 旋转动画流畅、不阻塞滚动

## 2. 种子测试点（TP 模板）

### TP-001（ANIMATION）：页面转场动画流畅
- **scenario**：场景 1
- **module**：`ANIMATION`
- **precondition**：游戏内
- **test_data**：连续切换 5 次页面
- **expected**：每次转场流畅、帧率 ≥ 30fps、无白屏
- **notes**：用 Performance 面板监测 FPS

### TP-002（ANIMATION）：弹窗入场动画
- **scenario**：场景 2
- **module**：`ANIMATION`
- **precondition**：触发弹窗
- **test_data**：点击触发 → 观察动画过程
- **expected**：弹窗从中心放大或从底部滑入、动画时长 200-300ms、不卡顿
- **notes**：注意动画曲线（ease-in / ease-out）

### TP-003（ANIMATION）：弹窗退场动画
- **scenario**：场景 2
- **module**：`ANIMATION`
- **precondition**：弹窗打开
- **test_data**：点击关闭 → 观察动画
- **expected**：弹窗缩小或滑出、与入场动画对应
- **notes**：中途关闭 vs 完整关闭

### TP-004（ANIMATION）：按钮点击反馈
- **scenario**：场景 3
- **module**：`ANIMATION`
- **precondition**：购买按钮可点
- **test_data**：按下按钮 → 释放
- **expected**：按下缩小（0.95x）、释放恢复、有触觉反馈（移动端）
- **notes**：与 A-009 配合（A 测响应、E 测反馈动画）

### TP-005（ANIMATION）：进度条平滑
- **scenario**：场景 4
- **module**：`ANIMATION`
- **precondition**：加载中
- **test_data**：观察进度条
- **expected**：进度条平滑前进（不是跳变）、与实际进度匹配
- **notes**：注意"假装进度" vs "真实进度"

### TP-006（ANIMATION）：飘字动画
- **scenario**：场景 5
- **module**：`ANIMATION`
- **precondition**：获得奖励
- **test_data**：观察飘字从道具位置飞向货币栏
- **expected**：飘字轨迹流畅、淡入淡出、不卡顿
- **notes**：游戏专项（与 HINT 模块有边界）

### TP-007（ANIMATION）：闪烁提示
- **scenario**：场景 6
- **module**：`ANIMATION`
- **precondition**：有新消息
- **test_data**：观察红点
- **expected**：红点闪烁频率合理（不刺眼）、点击后停止
- **notes**：与 F.引导/红点 配合

### TP-008（ANIMATION）：加载动画不阻塞
- **scenario**：场景 7
- **module**：`ANIMATION`
- **precondition**：下拉刷新
- **test_data**：下拉 → 观察 spinner
- **expected**：spinner 旋转流畅、不阻塞列表滚动
- **notes**：注意动画与主线程的解耦

### TP-009（ANIMATION）：动画不闪屏
- **scenario**：场景 1
- **module**：`ANIMATION`
- **precondition**：低帧率设备
- **test_data**：在低帧率下切换页面
- **expected**：动画不闪屏、不撕裂（按设计优雅降级）
- **notes**：与 J.游戏专项 配合

### TP-010（ANIMATION）：动画可中断
- **scenario**：场景 2
- **module**：`ANIMATION`
- **precondition**：弹窗入场动画中
- **test_data**：动画过程中点击关闭
- **expected**：动画立即中断、弹窗开始退场
- **notes**：注意动画状态管理

## 3. 边界陷阱

### 边界 1：vs A. 控件基础
- **混淆点**：「弹窗打开"后"状态"」——A 测打开后状态、E 测打开过程
- **判定规则**：测"打开/关闭后状态正确" → A；测"打开/关闭过程动画流畅" → E
- **实例**：弹窗打开后遮罩锁定 → A-009；弹窗打开动画是否卡顿 → E-002

### 边界 2：vs D. 静态展示
- **混淆点**：「加载占位"动效"」——D 测静态资源、E 测动画
- **判定规则**：测"占位图的样式" → D；测"loading spinner 动画" → E
- **实例**：占位图灰色块 → D-006；spinner 旋转 → E-008

### 边界 3：vs HINT
- **混淆点**：「飘字"动画"」——E 测动画过程、HINT 测内容
- **判定规则**：测"飘字轨迹流畅" → E；测"飘字内容/数字正确" → HINT
- **实例**：飘字飞行动画流畅 → E-006；飘字显示"获得 100 金币" → HINT

### 边界 4：vs LOG
- **混淆点**：「动画"完成"埋点」——E 测动画本身、LOG 测埋点
- **判定规则**：测"动画是否流畅" → E；测"动画完成事件上报" → LOG
- **实例**：动画帧率达标 → E-001；动画完成时上报 `animation_end` → LOG

## 4. 验证证据

### 视觉证据
- 动画过程截图序列（关键帧）
- 录屏文件

### 日志证据
- `animation.start/end` 事件
- 动画完成回调

### 数据证据
- Performance 面板：FPS ≥ 30
- 动画帧时间 < 33ms

### 性能证据
- 动画不阻塞主线程
- 动画过程 CPU 占用 < 30%
- 低端设备流畅度降级策略
