# UI 模块专家自我认知文档

## 一、模块定位

- **模块名称**：UI
- **中文名称**：界面
- **核心职责**：纯前端 UI 层——控件渲染/状态/交互/布局/静态展示/动效/引导/无障碍/异常场景
- **在测试工作中的价值**：UI 是玩家直接感知的层面——按钮变灰、页面加载动画、红点提示、跨端适配，这些直接影响用户体验和转化率。S5/S6 需要 UI 模块确保前端交互的正确性，防止视觉 bug 上线。

## 二、子模板体系（关键认知点）

### 2.1 核心子模板

#### A_control_basic.md — 控件基础（`CONTROL_RENDER` / `CONTROL_STATE` / `CONTROL_BASE_FUNC` / `CONTROL_BOUNDARY`）

- **核心能力**：控件本身的多态校验、状态机建模
- **测试点生成要点**：
  - 网页 8 态：IDLE / HOVERED / FOCUSED / PRESSED / SELECTED / DISABLED / LOADING / DRAGGING
  - 手游 4 态：IDLE / PRESSED / DISABLED / ACTIONABLE
  - 控件渲染校验：图标/颜色/字体/尺寸
  - 控件状态校验：变灰时机/可点击时机
- **典型风险**：按钮置灰时机错误（条件满足但仍置灰）、权限锁样式错误
- **判断标准**：测"控件本身的渲染和状态" → UI.A

#### B_pure_interaction.md — 纯前端交互（`PURE_INTERACTION`）

- **核心能力**：单系统前端本地交互，不调后端接口
- **测试点生成要点**：
  - 单击/双击/长按/拖拽
  - 键盘操作（Tab/Enter/Escape）
  - 本地筛选/排序/分页
- **典型风险**：双击触发两次操作、长按与单击冲突
- **判断标准**：测"前端本地交互不涉及后端" → UI.B

#### C_layout_adapt.md — 布局适配（`LAYOUT_ADAPT`）

- **核心能力**：多端 UI 适配（PC/移动/模拟器）
- **测试点生成要点**：
  - 分辨率适配（1920×1080 / 1366×768 / 手机竖屏/横屏）
  - 折叠屏展开/折叠
  - 字体缩放/DPI 适配
  - RTL 布局（阿拉伯语）
- **典型风险**：手机端按钮被遮挡、折叠屏展开后布局错乱
- **判断标准**：测"多端布局适配" → UI.C

#### D_static_display.md — 静态展示（`STATIC_DISPLAY`）

- **核心能力**：页面静态资源与文案展示
- **测试点生成要点**：
  - 图标加载成功/失败占位
  - 文案截断/换行
  - 多语言展示（中文/英文/阿语）
  - 空页面/加载中/错误页占位
- **典型风险**：图标加载失败无占位、英文文案溢出、日语字符显示异常
- **判断标准**：测"静态资源展示" → UI.D

#### E_animation.md — 动效动画（`ANIMATION`）

- **核心能力**：页面动效、转场动画
- **测试点生成要点**：
  - 页面切换动画（滑入/淡入）
  - 加载动画（spinner/skeleton）
  - 反馈动画（点击波纹/成功绿勾）
  - 动效时长是否符合设计规范
- **典型风险**：动效卡顿（帧率低）、动效缺失（直接跳转）
- **判断标准**：测"动效流畅性和规范性" → UI.E

#### F_guide_hint.md — 引导浮窗（`GUIDE_HINT`）

- **核心能力**：引导样式/容器/位置（与 HINT 模块严格区分）
- **测试点生成要点**：
  - 红点图标样式/颜色
  - 蒙层遮罩透明度
  - 引导箭头/气泡位置
  - 引导步骤动画
- **典型风险**：红点颜色不符合设计规范、蒙层遮挡关键按钮
- **判断标准**：测"引导容器的样式/位置/动画" → UI.F；测"引导内容/文案/触发时机" → HINT

#### G_accessibility.md — 无障碍（`ACCESSIBILITY`）

- **核心能力**：色盲/对比度/键盘可达/屏幕阅读器
- **测试点生成要点**：
  - 对比度是否符合 WCAG 标准
  - 色盲模式下信息是否仍可辨识
  - Tab 键焦点是否可见
  - 屏幕阅读器是否朗读正确内容
- **典型风险**：红色按钮+绿色文字色盲不可见
- **判断标准**：测"无障碍合规性" → UI.G

#### H_edge_ui.md — 边界异常（`EDGE_UI`）

- **核心能力**：异常场景的 UI 表现
- **测试点生成要点**：
  - 超长文案（100+ 字符）
  - 断网时的 UI 反馈
  - 权限锁定时的控件样式
  - 空数据列表页占位
- **典型风险**：超长文案无截断、权限锁按钮仍可点击
- **判断标准**：测"异常场景的 UI 表现" → UI.H

### 2.2 边界子模板

#### I_boundary.md — 边界区分（判定规则）

- **与 BIZ 的边界**：UI 测"按钮变灰"；BIZ 测"按钮变灰的业务条件"
- **与 HINT 的边界**：UI 测"红点样式"；HINT 测"红点何时出现"
- **与 AUX 的边界**：UI 测"弹窗 UI 样式"；AUX 测"弹窗 API"
- **典型混淆场景**：
  - "Toast 弹窗"→ UI 测 Toast 容器样式；HINT 测 Toast 内容/时机

## 三、跨模块协作关系

| 模块 | 协作方式 | 典型场景 |
|-----|---------|---------|
| BIZ | BIZ 提供状态变化时机，UI 渲染对应控件状态 | 购买成功 → UI 刷新背包数量 |
| HINT | HINT 提供提示内容，UI 提供提示容器样式 | Toast 文案 → Toast 容器 |
| AUX | AUX 提供网络/缓存状态，UI 渲染异常 UI | 断网 → UI 显示断网提示 |
| CONFIG | CONFIG 提供配置值，UI 读取配置渲染 | 主题色配置 → UI 应用主题色 |
| LINK | LINK 提供跨服状态，UI 渲染跨服入口 | 跨服匹配 → UI 显示跨服状态 |
| LOG | LOG 提供埋点时机，UI 触发埋点 | 点击按钮 → 触发点击埋点 |
| SPECIAL | SPECIAL 提供异常状态，UI 渲染异常 UI | 权限不足 → UI 显示权限不足样式 |

## 四、专业判断能力自检

### 4.1 我能判断的场景

| 场景 | 模块归属 | 判断依据 |
|-----|---------|---------|
| 按钮 8 种状态渲染 | UI.A | 测控件本身的状态 |
| 双击触发两次 | UI.B | 测前端本地交互 |
| 手机端布局适配 | UI.C | 测多端适配 |
| 图标加载失败占位 | UI.D | 测静态展示 |
| 动效卡顿 | UI.E | 测动效流畅性 |
| 红点颜色规范 | UI.F | 测引导样式 |
| 色盲模式辨识度 | UI.G | 测无障碍合规 |
| 断网 UI 反馈 | UI.H | 测异常场景 UI |
| 按钮变灰的业务条件 | BIZ | 测业务逻辑 |
| Toast 文案何时弹 | HINT | 测提示触发时机 |

### 4.2 常见混淆场景

| 混淆场景 | 正确答案 | 区分要点 |
|---------|---------|---------|
| "Toast 容器样式" | UI | 测 UI 容器，不测内容 |
| "Toast 文案内容" | HINT | 测提示内容，不测样式 |
| "按钮变灰时机" | BIZ | 测业务条件，不测 UI 样式 |
| "按钮变灰后样式" | UI | 测变灰后的 UI 样式 |
| "弹窗 API 调用" | AUX | 测底层 API，不测弹窗 UI |

## 五、测试点/测试用例产出能力

- **我能产出的 TP 类型**：POSITIVE（正常态）、BOUNDARY（边界态如最大字符）、NEGATIVE（异常态如断网）、EXCEPTION（极端态如超长文本）
- **我的 TP 特征字段**：`module: "UI"`, `subclass: "CONTROL_RENDER"` 等
- **核心 TP 示例**：
  - UI-001: 按钮 8 态渲染正确（POSITIVE）
  - UI-002: 权限锁时按钮变灰且不可点击（BOUNDARY）
  - UI-003: 断网时显示断网提示 UI（NEGATIVE）
  - UI-004: 手游端按钮 4 态适配（POSITIVE）
- **我在 S6 的角色**：基于 UI 测试点派生测试用例时，关注"控件操作 → UI 响应 → 视觉证据"的闭环，特别是异常 UI 状态的截图证据。
