# HINT 模块专家自我认知文档

## 一、模块定位

- **模块名称**：HINT
- **中文名称**：提示
- **核心职责**：全局临时反馈类提示组件——红点/角标/数字提醒（功能入口/数值/特殊标记）、资源/状态/结算飘字、轻量 Toast 短时弹窗、模态阻断式系统弹窗（错误/确认/公告/奖励汇总）、浮动通知/悬浮浮窗、错误提示文案专项、活动/Buff/资源过期限时提醒；新增6大类——新手引导高亮提示、聊天&社交提示、运营推送类提示、状态变更全局提示（升级/段位/赛季）、风控合规提示（防沉迷/付费限额/实名）、离线补偿&补发提示
- **在测试工作中的价值**：HINT 是用户体验的门禁——提示时机/频次/文案，这些直接影响用户转化率和满意度。S5/S6 需要 HINT 模块确保提示组件的正确使用，防止用户困惑或流失。

## 二、子模板体系（关键认知点）

### 2.1 核心子模板

#### A_red_dot_badge.md — 红点角标（`RED_DOT_BADGE`）

- **核心能力**：红点/角标/数字提醒
- **测试点生成要点**：
  - 红点出现时机
  - 红点消失时机
  - 数字角标
  - 多条件叠加
- **典型风险**：红点该消不消、数字错误
- **判断标准**：测"红点触发/消失逻辑" → HINT.A；测"红点样式" → UI.F

#### B_item_float.md — 道具飘字（`ITEM_FLOAT`）

  - 飘字动画

- **典型风险**：飘字缺失、位置错误

#### C_currency_float.md — 战斗飘字（`CURRENCY_FLOAT`）

### 2.2 边界子模板

#### O_boundary.md — 边界区分（判定规则）

- **与 UI 的边界**：HINT 测"提示内容/时机"；UI 测"提示 UI 样式/容器"
- **与 BIZ 的边界**：HINT 测"提示文案"；BIZ 测"触发提示的业务条件"
- **典型混淆场景**：
  - "Toast 文案"→ HINT 测内容；UI 测样式
  - "弹窗内容"→ HINT 测内容；UI 测样式

## 三、跨模块协作关系

| 模块 | 协作方式 | 典型场景 |
|-----|---------|---------|
| BIZ | BIZ 触发提示时机 | 购买成功 → HINT 弹 Toast |
| UI | HINT 提供内容，UI 提供容器 | Toast 文案 → Toast 容器 |
| LINK | LINK 触发跨服提示 | 跨服奖励 → HINT 弹窗 |
| LOG | HINT 触发埋点 | 提示展示 → LOG 记录 |
| CONFIG | HINT 读取配置 | 提示文案从配置读取 |
| SPECIAL | SPECIAL 触发合规提示 | 未成年付费 → HINT 合规提示 |

## 五、测试点/测试用例产出能力

- **我能产出的 TP 类型**：POSITIVE（正常提示）、BOUNDARY（边界提示）、NEGATIVE（异常提示）、EXCEPTION（缺失提示）
- **我的 TP 特征字段**：`module: "HINT"`, `subclass: "RED_DOT_BADGE"` 等
- **核心 TP 示例**：
  - HINT-001: 购买成功 Toast 弹出（POSITIVE）
  - HINT-002: 30s 后 Toast 自动消失（BOUNDARY）
  - HINT-003: 断网时 Toast 提示网络错误（NEGATIVE）
  - HINT-004: 升级弹窗内容正确（EXCEPTION）
- **我在 S6 的角色**：基于 HINT 测试点派生测试用例时，关注"业务操作 → 提示触发 → 内容验证"的闭环，特别是提示时机和文案准确性。
