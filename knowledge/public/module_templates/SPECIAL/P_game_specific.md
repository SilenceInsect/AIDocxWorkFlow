# P. SPECIAL 模块游戏项目补充

> **仅游戏项目**——非游戏项目可忽略本文件。
>
> 本文件是 S5 生成 SPECIAL 模块测试点时的**游戏项目专项补充**，
> 覆盖 5 个游戏子领域：战斗/社交/经济/活动/防沉迷/赛事。
> 每个子领域提供典型场景 + 种子 TP + 边界陷阱。

---

## P-1. 战斗相关

### 场景 1：战斗中断线
- 业务背景：玩家在副本战斗中断线
- 涉及字段/工具：battle_state、reconnect
- 触发动作：断线 30s 后重连
- 验证点：战斗数据不丢 + 可继续战斗

### 场景 2：战斗切后台
- 业务背景：战斗过程中切后台
- 涉及字段/工具：battle_pause、bg_event
- 触发动作：切后台 1min
- 验证点：返回前台战斗继续 + 无数据错乱

### 场景 3：战斗资源耗尽
- 业务背景：战斗时技能蓝量耗尽
- 涉及字段/工具：mp_pool、resource_check
- 触发动作：玩家释放技能
- 验证点：技能释放失败 + 提示"蓝量不足"

### 场景 4：PVP 作弊
- 业务背景：PVP 中检测到玩家瞬移
- 涉及字段/工具：pvp_position、cheat_detect
- 触发动作：玩家使用外挂
- 验证点：检测到作弊 + 封禁 + PVP 战斗回滚

### 场景 5：PVP 断线重连
- 业务背景：PVP 中断线
- 涉及字段/工具：pvp_reconnect、match_state
- 触发动作：PVP 中断线 10s
- 验证点：重连后继续 PVP + 对方等待超时

#### TP-001（P-战斗）：副本战斗断线恢复
- **module**：`BOUNDARY_EXTREME`（SPECIAL 子类）
- **precondition**：玩家在副本战斗中
- **test_data**：断网 30s → 重连
- **expected**：战斗数据不丢（血量/BUFF/位置） + 可继续
- **notes**：注意"断线" vs "退出"——断线需重连恢复

#### TP-002（P-战斗）：PVP 作弊封禁
- **module**：`ANTI_CHEAT`（SPECIAL 子类）
- **precondition**：PVP 中检测到瞬移
- **test_data**：玩家使用外挂瞬移
- **expected**：检测作弊 + 战斗回滚 + 封禁
- **notes**：注意"PVP" vs "PVE"——PVP 作弊影响对方

## P-2. 社交相关

### 场景 1：私聊敏感词
- 业务背景：玩家发送敏感词
- 涉及字段/工具：private_msg、sensitive_filter
- 触发动作：发送"违禁词"
- 验证点：消息拦截 + 警告

### 场景 2：好友作弊
- 业务背景：玩家通过外挂批量加好友
- 涉及字段/工具：add_friend、batch_detect
- 触发动作：脚本 1min 加 1000 好友
- 验证点：限流 + 检测 + 封号

### 场景 3：聊天刷屏
- 业务背景：玩家刷屏聊天
- 涉及字段/工具：chat_rate、spam_filter
- 触发动作：1s 发 100 条消息
- 验证点：限流 + 警告 + 禁言

### 场景 4：工会资金风控
- 业务背景：工会资金被异常转移
- 涉及字段/工具：guild_fund、risk_check
- 触发动作：工会管理员转移 100 万金币
- 验证点：风控拦截 + 二次确认 + 日志

#### TP-003（P-社交）：聊天刷屏禁言
- **module**：`WEAK_NET_RATE_LIMIT`（SPECIAL 子类）
- **precondition**：玩家登录
- **test_data**：1s 发 100 条消息
- **expected**：限流 + 警告 + 禁言 1h
- **notes**：注意"刷屏" vs "高频点击"——前者是消息

#### TP-004（P-社交）：工会资金大额风控
- **module**：`SERVER_HA_RISK`（SPECIAL 子类）
- **precondition**：工会资金 100 万
- **test_data**：转移 100 万金币
- **expected**：风控拦截（> 50 万需上级审批）+ 二次确认 + 日志
- **notes**：注意"工会" vs "个人"——工会资金风控更严

## P-3. 经济相关

### 场景 1：拍卖行并发抢拍
- 业务背景：1 件商品 1000 人抢拍
- 涉及字段/工具：auction_lock、bid_order
- 触发动作：1000 个并发 bid
- 验证点：分布式锁 + 按出价排序 + 1 人成交

### 场景 2：交易所套利风控
- 业务背景：玩家利用漏洞套利
- 涉及字段/工具：trade_pattern、abuse_detect
- 触发动作：玩家异常交易
- 验证点：检测套利 + 封号 + 资源回收

### 场景 3：充值大额风控
- 业务背景：玩家一次性充值 10 万
- 涉及字段/工具：large_pay、risk_check
- **expected**：单笔上限拦截 + 人工审核 + 二次确认

### 场景 4：游戏币来源异常
- 业务背景：玩家金币来源异常（如凭空多出 100 万）
- 涉及字段/工具：gold_source、source_check
- 触发动作：检测到金币来源异常
- 验证点：追溯来源 + 异常金币回收

#### TP-005（P-经济）：拍卖行并发分布式锁
- **module**：`SERVER_HA_RISK`（SPECIAL 子类）
- **precondition**：1 件商品
- **test_data**：1000 个并发 bid
- **expected**：Redis 分布式锁 + 按出价时间/价格排序 + 1 人成交 + 无超卖
- **notes**：注意"并发" vs "顺序"——并发需锁

#### TP-006（P-经济）：游戏币来源异常回收
- **module**：`ANTI_CHEAT`（SPECIAL 子类）
- **precondition**：玩家金币凭空多出 100 万
- **test_data**：审计触发
- **expected**：追溯来源 + 异常金币回收 + 日志
- **notes**：注意"异常" vs "活动奖励"——区分合法/非法

## P-4. 活动相关

### 场景 1：活动未开启拦截
- 业务背景：限时活动 19:00 开启
- 涉及字段/工具：activity_start、claim_check
- 触发动作：18:59 玩家点击"领取"
- 验证点：拒绝 + 提示"活动未开启"

### 场景 2：活动重复领奖
- 业务背景：玩家尝试重复领奖
- 涉及字段/工具：claim_count、dedup
- 触发动作：玩家脚本批量领奖
- 验证点：去重 + 限流 + 封号

### 场景 3：活动容量满
- 业务背景：活动 1 万名额满
- 涉及字段/工具：activity_capacity
- 触发动作：玩家尝试进入
- 验证点：拒绝 + 提示"活动已满"

### 场景 4：活动跨服同步
- 业务背景：跨服活动数据不同步
- 涉及字段/工具：cross_server_activity、sync
- 触发动作：玩家跨服查看活动
- 验证点：数据同步 + 一致性

#### TP-007（P-活动）：活动并发抢进
- **module**：`BOUNDARY_EXTREME`（SPECIAL 子类）
- **precondition**：活动 1 万名额
- **test_data**：1min 内 10 万人并发 enter
- **expected**：队列 + 限流 + 1 万人成功 + 9 万排队/拒绝
- **notes**：注意"并发" vs "顺序"——并发需分布式锁

#### TP-008（P-活动）：活动跨服同步
- **module**：`CHANNEL_GRAY_BIZ`（SPECIAL 子类）
- **precondition**：跨服活动进行中
- **test_data**：A 服玩家查看活动进度
- **expected**：进度与 B 服一致 + 数据同步
- **notes**：注意"跨服" vs "本服"——跨服需同步

## 边界陷阱（游戏专项）

### 边界 1：vs BIZ（游戏业务）
- **混淆点**："战斗"看似 BIZ → 实际 BIZ 测"战斗正常流程"，SPECIAL 测"战斗异常/作弊/资源耗尽"
- **判定规则**：测"战斗正常" → 归 BIZ；测"战斗异常/对抗/极限" → 归 SPECIAL
- **instance**：正常战斗伤害 → 归 BIZ；战斗作弊检测 → 归 SPECIAL

### 边界 2：vs CONFIG（游戏配置）
- **混淆点**："活动配置"看似 CONFIG → 实际 CONFIG 测"活动配置字段"，SPECIAL 测"活动异常处理"
- **判定规则**：测"活动配置" → 归 CONFIG；测"活动异常（未开启/容量满/重复领奖）" → 归 SPECIAL
- **instance**：活动时间配置 → 归 CONFIG；活动未开启拦截 → 归 SPECIAL

### 边界 3：vs LINK（跨服）
- **混淆点**："跨服活动"看似 LINK → 实际 LINK 测"跨服数据同步"，SPECIAL 测"跨服活动异常"
- **判定规则**：测"跨服数据正常同步" → 归 LINK；测"跨服活动异常（同步失败/作弊拦截）" → 归 SPECIAL
- **instance**：跨服组队 → 归 LINK；跨服活动同步失败 → 归 SPECIAL

### 边界 4：vs LOG（游戏审计）
- **混淆点**："赛事奖励"看似 LOG → 实际 LOG 测"奖励日志"，SPECIAL 测"奖励并发拦截"
- **判定规则**：测"奖励日志完整性" → 归 LOG；测"奖励并发去重/限流" → 归 SPECIAL
- **instance**：奖励日志查询 → 归 LOG；奖励并发去重 → 归 SPECIAL
