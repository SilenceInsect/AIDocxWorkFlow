# O. HINT 模块边界区分（判定规则）

> **非测试类型**——本文件是 S5 生成测试点时的**判定参考**，
> 避免误标 HINT 标签（实际是 UI / BIZ / UTIL / LINK / LOG / SPECIAL / CONFIG）。
>
> **来源**：`.cursor/MODULES.md` §4.11 O

---

## 1. 核心判定原则

> **HINT 聚焦"玩家可见的全局临时反馈类提示组件"**；页面常驻控件、固定布局、静态页面元素**统归 UI**。

### 1.1 关键边界隔离规则（S5 误标高发区）

> **核心判断三问**（S5 prompt 强制回答）：
> 1. **该元素是事件触发弹出，还是页面常驻可见？** 事件触发 → HINT；常驻 → UI
> 2. **该元素是操作后自动消失，还是玩家手动关闭/长期保留？** 自动消失 → HINT；长期保留 → UI
> 3. **该测试点是测"显示什么内容/数字/文案"（HINT），还是测"UI 容器的样式/位置/动画"（UI F.GUIDE_HINT）？** 内容 → HINT；样式 → UI

| 归 HINT（不归 UI）                                                                                          | 归 UI（不归 HINT）                                                                       |
| ----------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| 临时弹出、一次性反馈、全局浮动、操作后自动消失的提示组件                                                  | 页面常驻控件、固定布局、页面内置按钮/输入框/分页                                          |
| 红点、飘字、Toast、弹窗、浮窗、引导气泡                                                                    | 静态展示、页面内置文字标签、页面内常驻数值显示                                            |
| **举例**：背包页面里**固定显示的金币数字** → UI；使用道具弹出**金币+100 飘字** → HINT                      | **举例**：活动页面**常驻活动标题** → UI；上线自动弹出**活动奖励弹窗** → HINT             |
| 邮件未读数角标（**触发弹性的**）                                                                           | 邮件页面固定列表/分页器/搜索框                                                            |
| 战斗中暴击伤害飘字（**一次性浮现后消失**）                                                                 | 战斗血条/角色模型/技能图标（**常驻渲染**）                                                |
| 限时活动倒计时浮窗（**事件触发弹出**）                                                                     | 活动页面常驻倒计时数字（**页面常驻**）                                                    |
| 升级弹窗、突破升星弹窗（**事件触发**）                                                                     | 升级页面常驻控件（**页面常驻**）                                                          |
| 新手引导高亮遮罩（**事件触发、点击后消失**）                                                                | 引导底层框架（步骤管理器）→ UTIL                                                          |
| 错误提示文案分级样式（**红黄绿配色**）                                                                     | 错误码体系、错误处理业务逻辑 → BIZ                                                       |

## 2. 边界对照表

### 2.1 vs UI（界面）—— **最易混淆**

| 场景                                | 归 HINT（内容/触发/文案）                          | 归 UI（样式/位置/动画）                            |
| ----------------------------------- | -------------------------------------------------- | ------------------------------------------------- |
| 红点                                | 红点显示/隐藏的逻辑、是否已读                      | 红点图标位置/大小/闪烁动画                          |
| Toast                                | Toast 显示什么文字（成功/失败/警告）                | Toast 弹窗位置/时长/动画                            |
| 飘字                                | 飘字显示什么数字（+100 金币）                       | 飘字轨迹/动画/淡入淡出                              |
| 系统弹窗                            | 弹窗内容（"确定要退出吗"）                         | 弹窗 UI 样式/层级/关闭                              |
| 倒计时                              | 倒计时逻辑（与服务器时间同步）                     | 倒计时 UI 位置/刷新策略                            |
| 引导                                | 引导内容（"点这里领取奖励"）/步骤逻辑              | 引导遮罩/高亮/气泡 UI 样式                          |
| 战斗飘字                            | 飘字显示"+9999" 内容/暴击/治疗文字                | 飘字颜色/放大动画                                  |
| 聊天消息                            | 私聊红点显示/清除/弹窗                             | 聊天 UI 渲染/表情包/输入框                          |
| 升级弹窗                            | 升级文案/新等级/特效                                | 升级弹窗 UI 样式                                    |
| 活动入口                            | 活动侧边浮窗显示/触发                               | 活动按钮 UI/图标                                    |

## 3. 常见误判案例

### 误判 1：Toast "购买成功" → 错标 UI

**错误**：看到 Toast 弹窗就标 UI
**正确**：
- UI 测"Toast 弹窗的 UI 样式/位置/时长"（归 UI `F.GUIDE_HINT`）
- HINT 测"Toast 显示什么内容、什么时机触发"（归 HINT E `TOAST`）

### 误判 2：红点 → 错标 UI

**错误**：看到红点就标 UI
**正确**：
- UI 测"红点 UI 图标/位置/闪烁"（归 UI `F.GUIDE_HINT`）
- HINT 测"红点显示/隐藏的逻辑、是否已读"（归 HINT A `RED_DOT_BADGE`）

### 误判 3：飘字 +100 → 错标 UI

**错误**：看到飘字就标 UI
**正确**：
- UI 测"飘字轨迹/动画/淡入淡出"（归 UI `E.ANIMATION`）
- HINT 测"飘字显示什么数字"（归 HINT B `ITEM_FLOAT` 或 C `CURRENCY_FLOAT`）

### 误判 4：升级弹窗 → 错标 BIZ

**错误**：看到"升级"就标 BIZ
**正确**：
- BIZ 测"升级状态机、经验计算"（归 BIZ `D.STATE_MACHINE`）
- HINT 测"升级弹窗显示/文案/特效"（归 HINT K `STATE_CHANGE_DIALOG`）

### 误判 5：登录失败弹窗 → 错标 UI

**错误**：看到登录弹窗就标 UI
**正确**：
- UI 测"登录弹窗 UI 样式"（归 UI `A.CONTROL_BASE_FUNC`）
- HINT 测"登录失败错误弹窗"（归 HINT D `MODAL_DIALOG`）
- BIZ 测"登录逻辑、token 存储"（归 BIZ `A.BIZ_LOGIC`）
- LINK 测"调第三方登录"（归 LINK `D.EXTERNAL_THIRD_PARTY`）

### 误判 6：防沉迷弹窗 → 错标 SPECIAL

**错误**：看到"防沉迷"就标 SPECIAL
**正确**：
- SPECIAL 测"防沉迷规则/未成年检测"（归 SPECIAL `H.COMPLIANCE_RISK`）
- HINT 测"防沉迷弹窗显示/文案"（归 HINT L `COMPLIANCE_PROMPT`）

### 误判 7：新手引导 → 错标 UI

**错误**：看到引导就标 UI
**正确**：
- UI 测"引导遮罩/高亮/气泡 UI 样式"（归 UI `F.GUIDE_HINT`）
- HINT 测"引导内容/步骤/触发逻辑"（归 HINT H `GUIDE_HIGHLIGHT`）
- UTIL 测"引导底层框架（步骤管理器）"（归 UTIL）

### 误判 8：离线奖励 → 错标 BIZ

**错误**：看到"离线奖励"就标 BIZ
**正确**：
- BIZ 测"离线奖励计算业务"（归 BIZ `A.BIZ_LOGIC`）
- HINT 测"离线奖励弹窗显示"（归 HINT M `OFFLINE_COMPENSATION`）

## 4. 判定流程图（S5 prompt 决策树）

```
Story 涉及某个功能点
    │
    ├── 该元素是页面常驻控件 / 固定布局 / 静态展示？
    │     是 → 归 UI
    │
    ├── 该元素是事件触发弹出 / 临时反馈 / 自动消失？
    │     是 → 归 HINT
    │     │
    │     ├── 涉及后端接口/数据/状态？
    │     │     是 → 配 BIZ 测试（业务逻辑）
    │     │
    │     ├── 涉及外部系统（支付/登录/推送）？
    │     │     是 → 配 LINK 测试
    │     │
    │     ├── 涉及网络/资源/缓存？
    │     │     是 → 配 UTIL 测试
    │     │
    │     ├── 涉及埋点/日志？
    │     │     是 → 配 LOG 测试
    │     │
    │     ├── 涉及异常处理/合规检测？
    │     │     是 → 配 SPECIAL 测试
    │     │
    │     ├── 涉及配置读取？
    │     │     是 → 配 CONFIG 测试
    │     │
    │     └── 是 HINT 内部哪个子类？
    │           ├── 红点/角标 → A
    │           ├── 资源飘字 → B
    │           ├── 战斗飘字 → C
    │           ├── 模态弹窗 → D
    │           ├── Toast → E
    │           ├── 浮动通知 → F
    │           ├── 限时提醒/错误文案 → G
    │           ├── 新手引导 → H
    │           ├── 社交提示 → I
    │           ├── 运营推送 → J
    │           ├── 状态变更 → K
    │           ├── 风控合规 → L
    │           └── 离线补偿 → M
    │
    └── 判定：HINT 测"内容/触发/文案"，UI 测"样式/位置/动画"
```

## 5. HINT vs UI F.GUIDE_HINT 关键区分

> **最重要的边界**：HINT 测内容、UI F.GUIDE_HINT 测样式。

| 维度              | HINT（内容）| UI F.GUIDE_HINT（样式）|
| ----------------- | ----------- | ---------------------- |
| 测什么            | 显示什么文字/数字/逻辑/触发 | 位置/大小/动画/时长/颜色/层级 |
| 例子              | Toast"购买成功"、飘字"+100" | Toast 顶部居中 2s、飘字淡入 0.5s |
| 归类依据          | 内容/触发/文案/业务规则 | UI 容器/样式/动画 |

**判例**：

| 实际场景                                       | 归 HINT         | 归 UI             |
| ---------------------------------------------- | --------------- | ----------------- |
| 购买成功 Toast                                 | E.TOAST（内容）| F.GUIDE_HINT（样式）|
| 红点触发清除                                   | A.RED_DOT_BADGE | F.GUIDE_HINT      |
| 暴击伤害飘字                                   | C.CURRENCY_FLOAT | E.ANIMATION      |
| 战斗 buff 图标常驻                             | —              | F.GUIDE_HINT      |
| 战斗 buff 获得瞬间飘字                         | C.CURRENCY_FLOAT | —                |
| 新手引导高亮遮罩                               | H.GUIDE_HIGHLIGHT（内容）| F.GUIDE_HINT（样式）|
| 错误提示文案分级                               | G.TIMED_REMINDER（文案）| F.GUIDE_HINT（样式）|
