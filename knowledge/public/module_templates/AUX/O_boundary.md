# O. AUX 模块边界区分（判定规则）

> **非测试类型**——本文件是 S5 生成测试点时的**判定参考**，
> 避免误标 AUX 标签（实际是 BIZ / CONFIG / UI / LOG / HINT / SPECIAL / LINK）。
>
> **v1.6.1 变更**：AUX 收窄为"底层基础工具"——高层业务辅助（日志/提示/第三方/风控/业务异常）已迁出至其他模块。

---

## 1. 判定原则

> **AUX = 底层通用基础能力 + 工具 + 框架组件**
>
> **不归 AUX**：业务流程、玩家交互逻辑、配置表字段、页面视觉、业务协议的业务逻辑、高层业务辅助（提示/日志/第三方/风控）

### 判定三问

1. **是"通用工具/框架底层"还是"业务逻辑"？** 是业务 → 不归 AUX
2. **是"底层能力"还是"上层功能/业务辅助"？** 是上层功能 → 不归 AUX
3. **是"通用技术底层"还是"业务场景"？** 是业务场景 → 不归 AUX

## 2. 边界对照表

### vs BIZ（业务）

| 场景 | 归 AUX | 归 BIZ |
|------|--------|--------|
| 网络层 | 断线重连、协议打包、心跳 | 业务协议字段、业务流程 |
| 缓存层 | Redis 读写、缓存击穿 | 缓存策略（排行榜 5min）|
| GM 工具 | GM 指令、权限 | 业务发放逻辑（发 100 钻石）|
| 汇率换算 | 1 元 = 10 钻石换算 | 充值扣款业务 |
| 资源管理 | 资源加载/释放 | 业务用资源（如 UI 用图标）|
| 异常兜底 | 全局崩溃捕获、错误上报 | 业务异常处理（如购买失败补偿）|
| 加密 | 加密算法、签名 | 业务风控（账号风控）|
| 性能 | FPS 监控、内存监控 | 业务性能需求 |

### vs CONFIG

| 场景 | 归 AUX | 归 CONFIG |
|------|--------|-----------|
| 热更新 | 资源/客户端版本热更 | 配置文件热更（道具表）|
| 导出/导入 | — | Excel → JSON 导出工具 |
| 资源路径 | — | 资源路径字段 |
| 灰度 | 灰度玩家白名单工具 | 灰度配置字段 |
| 备份 | — | 配置备份工具 |

### vs UI

| 场景 | 归 AUX | 归 UI |
|------|--------|-------|
| 弹窗 | 弹窗公共封装（API）| 弹窗显示、样式 |
| 红点 | 全局红点 API | 红点 UI 渲染 |
| 路由 | 跳转路由 API | 页面渲染 |
| 加载动画 | 加载工具 | 加载动画 UI |
| 性能工具 | FPS 监控组件 | 性能 UI 显示 |

### vs LOG

| 场景 | 归 AUX | 归 LOG |
|------|--------|--------|
| 日志/埋点 | 日志/埋点工具、采集 SDK | 业务埋点（如购买埋点）|
| 崩溃 | 崩溃堆栈捕获 | 崩溃事件上报（业务层）|
| 审计 | GM 操作审计（GM 是 AUX）| 玩家行为审计 |

### vs HINT

| 场景 | 归 AUX | 归 HINT |
|------|--------|--------|
| 提示工具 | 弹窗 API | 提示内容/状态 |
| 红点 | 红点 API | 红点状态逻辑 |
| Toast | Toast API | Toast 内容/触发 |
| 飘字 | 飘字 API | 飘字内容 |

### vs SPECIAL

| 场景 | 归 AUX | 归 SPECIAL |
|------|--------|--------|
| 异常兜底 | 全局异常框架 | 业务异常（弱网降级）|
| 加密 | 加密算法 | 反作弊检测 |
| 网络 | 通用网络层 | 弱网/丢包/切后台 |

### vs LINK

| 场景 | 归 AUX | 归 LINK |
|------|--------|--------|
| 网络层 | 通用网络层（断线重连）| 跨服务调用、第三方支付 |
| 协议 | 协议打包解包 | 业务协议字段 |
| 同步 | 多端日志同步 | 多端数据同步 |

## 3. 常见误判案例

### 误判 1：抽卡概率 → 错标 AUX
- **错误**：看到"概率计算"，就标 AUX
- **正确**：
  - AUX 测"概率计算工具"（归 A-016）
  - CONFIG 测"概率配置总和 ≤ 100%"（归 CONFIG-G）
  - BIZ 测"抽卡逻辑/抽卡结果"（归 BIZ）
  - 同一 Story 应有 3 个 TP

### 误判 2：商城购买 → 错标 AUX
- **错误**：看到"商城扣款"，就标 AUX
- **正确**：
  - AUX 测"汇率换算"（归 E）
  - BIZ 测"购买流程/扣款/发货"（归 BIZ）
  - LINK 测"调第三方支付"（归 LINK）

### 误判 3：网络断开 → 错标 AUX
- **错误**：看到"断线"，就标 AUX
- **正确**：
  - AUX 测"断线重连"（归 B）
  - BIZ 测"断线时业务处理"（归 BIZ）
  - SPECIAL 测"弱网降级"（归 SPECIAL）
  - N 测"断线操作缓存"（归 N）

### 误判 4：活动开启 → 错标 AUX
- **错误**：看到"定时活动"，就标 AUX
- **正确**：
  - CONFIG-I 测"活动起止时间字段"（归 CONFIG）
  - BIZ 测"活动开启业务逻辑"（归 BIZ）
  - L 测"活动配置后台"（归 L）

### 误判 5：玩家数据 → 错标 AUX
- **错误**：看到"玩家数据存储"，就标 AUX
- **正确**：
  - AUX 测"通用 localStorage 工具"（归 A）
  - J 测"玩家设置存档"（归 J）
  - BIZ 测"业务数据 DB 存储"（归 BIZ）
  - C 测"运行时缓存"（归 C）

## 4. 判定流程图

```
Story 涉及某个功能点
    │
    ├── 是"通用工具/组件"？
    │     是 → 归 AUX（按子类 A-N 选择）
    │
    ├── 是"网络/缓存/资源底层"？
    │     是 → 归 AUX（按子类 B/C/D 选择）
    │
    ├── 是"GM/测试/运营工具"？
    │     是 → 归 AUX（按子类 G/H/I/L 选择）
    │
    ├── 是"性能/加密/异常兜底"？
    │     是 → 归 AUX（按子类 K/M/N 选择）
    │
    ├── 是"业务流程"？
    │     是 → 归 BIZ
    │
    ├── 是"配置表字段"？
    │     是 → 归 CONFIG
    │
    ├── 是"UI 渲染"？
    │     是 → 归 UI
    │
    ├── 是"跨服务/外部"？
    │     是 → 归 LINK
    │
    ├── 是"埋点/审计"？
    │     是 → 归 LOG
    │
    ├── 是"提示内容"？
    │     是 → 归 HINT
    │
    └── 是"特殊异常"？
          是 → 归 SPECIAL
```
