# B. 同表 / 全局一致性校验

> **子类代码**：`FIELD_INTRA_DEP`
> **归属模块**：`CONFIG`
> **来源**：用户细化定义 §2「配置一致性校验」（原项细化）
>
> **测什么**：同一表内多处属性数值统一、全表引用统一、环境一致性。
> **不测什么**：单字段合法性（归 A）、跨表 ID 引用（归 C）、热更（归 D）。
> **与其他子类的差异**：B 关注"同表内/全表/环境的统一"，A 关注"字段本身"，C 关注"跨表关联"。

---

## 1. 典型场景

### 场景 1：同道具多处属性统一
- 业务背景：道具表
- 涉及字段：`atk` / `def` / `hp`（同一道具 3 个字段）
- 触发动作：道具 A 攻击力 100，但防御力填了 99999（异常值）
- 验证点：同道具多字段值匹配合理范围

### 场景 2：同等级分段属性平滑
- 业务背景：角色等级表
- 涉及字段：1 级、2 级、3 级... 的属性
- 触发动作：1 级 HP=100，2 级 HP=10000，3 级 HP=10500
- 验证点：相邻等级属性变化应平滑，无跳变

### 场景 3：通用常量配置统一
- 业务背景：基础倍率表
- 涉及字段：`base_attack_rate`、`base_drop_rate`、`base_growth_rate`
- 触发动作：基础攻击力倍率在表 1 写 1.0，表 2 写 1.5
- 验证点：通用常量全表统一

### 场景 4：单位规范
- 业务背景：伤害表
- 涉及字段：`damage_value`
- 触发动作：道具 A 伤害 100，伤害单位是"点"；道具 B 伤害 100，伤害单位是"百分比"
- 验证点：全表伤害单位一致

### 场景 5：环境一致性（测试服/正式服）
- 业务背景：基础配置表
- 涉及字段：基础常量
- 触发动作：测试服基础攻击倍率 2.0，正式服 1.0
- 验证点：测试服/正式服基础常量基准一致（按设计）

### 场景 6：新旧版本兼容
- 业务背景：旧版本配置 + 新增字段
- 触发动作：旧客户端读取新版本配置（多了 `new_field` 字段）
- 验证点：旧客户端不报错，忽略新字段

### 场景 7：灰度/活动临时配置
- 业务背景：活动临时配置
- 涉及字段：活动期间临时上调掉率
- 触发动作：活动结束后，活动配置未清理
- 验证点：活动配置不污染全局基础表

## 2. 种子测试点（TP 模板）

### TP-001（FIELD_INTRA_DEP）：同道具多字段合理性
- **scenario**：场景 1
- **module**：`FIELD_INTRA_DEP`
- **precondition**：道具 A
- **test_data**：`atk=100, def=99999, hp=50`
- **expected**：导出工具检测到防御力异常，提示"超出合理范围"
- **notes**：合理范围按策划配置

### TP-002（FIELD_INTRA_DEP）：同道具数值一致性

## 3. 边界陷阱

### 边界 1：vs A. 字段合法性
- **混淆点**：「同道具多字段值"」——A 测单字段、B 测多字段一致
- **判定规则**：测"单字段值合法" → A；测"同表/全表多字段值统一" → B
- **实例**：攻击力字段类型 int → A-001；同道具攻击力 vs 防御力梯度 → B-001

### 边界 2：vs C. 跨表依赖
- **混淆点**：「表与表数值一致"」——B 测同表/全表、C 测跨表
- **判定规则**：测"单表内多字段一致" → B；测"表 A 字段值 = 表 B 字段值" → C
- **实例**：道具表攻击力全表统一 → B-005；任务表奖励道具 ID 存在于道具表 → C-001

### 边界 3：vs F. 版本兼容
- **混淆点**：「新旧版本"兼容"」——B 测数值一致、F 测版本兼容
- **判定规则**：测"同版本/同表内数值一致" → B；测"新/旧版本/新/旧客户端解析兼容" → F
- **实例**：同表 5 处基础倍率值一致 → B-005；旧客户端读新配置不报错 → F-001

### 边界 4：vs K. 游戏专项
- **混淆点**：「活动配置"清理"」——B 测不污染、K 测游戏专项
- **判定规则**：测"活动配置不污染全局" → B；测"活动配置过期自动失效/重置" → K
- **实例**：活动配置不污染基础表 → B-012；活动过期配置自动失效 → K-002
